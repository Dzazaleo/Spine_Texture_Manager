<objective>
Build the headless Spine loader. Given a path to a `.json` skeleton, it parses both the skeleton and its sibling `.atlas` file through the official `@esotericsoftware/spine-core` parsers, using a stub `TextureLoader` so no PNG bytes are touched. Returns a structured `LoadResult` containing `SkeletonData`, the parsed `TextureAtlas`, and a map of per-region source dimensions.

Purpose: the sampler needs `SkeletonData` + source dims, and the CLI needs to format atlas-not-found errors cleanly. This is the only module in Phase 0 that touches the filesystem.
Output: `src/core/loader.ts`, `src/core/types.ts`, `src/core/errors.ts`.
</objective>

// SkeletonJson parser
class SkeletonJson {
  constructor(attachmentLoader: AttachmentLoader);
  scale: number; // default 1
  readSkeletonData(json: string | object): SkeletonData;
}

// AttachmentLoader impl for atlases
class AtlasAttachmentLoader implements AttachmentLoader {
  constructor(atlas: TextureAtlas);
}

// Atlas parser — takes atlas text + a TextureLoader callback
// The constructor signature we use:
//   new TextureAtlas(atlasText: string)
// and then we manually assign pages' texture via atlas.setTextures() or by
// constructing pages with a TextureLoader. In spine-core 4.2 the canonical
// pattern is:
//   const atlas = new TextureAtlas(atlasText);
//   for (const page of atlas.pages) {
//     page.setTexture(fakeTextureFactory(page.name));
//   }
class TextureAtlas {
  pages: TextureAtlasPage[];
  regions: TextureAtlasRegion[];
  constructor(atlasText: string);
  findRegion(name: string): TextureAtlasRegion | null;
}

class TextureAtlasRegion /* extends TextureRegion */ {
  name: string;
  page: TextureAtlasPage;
  // Packed region in the atlas:
  x: number; y: number;
  width: number; height: number;        // packed dims
  // Original (pre-pack) size — what we want for source dims:
  originalWidth: number;
  originalHeight: number;
  offsetX: number;
  offsetY: number;
  degrees: number;
  u: number; v: number; u2: number; v2: number;
}

// Stub texture interface — we fabricate these, they are never decoded.
abstract class Texture {
  abstract dispose(): void;
  abstract setFilters(min: TextureFilter, mag: TextureFilter): void;
  abstract setWraps(uWrap: TextureWrap, vWrap: TextureWrap): void;
  getImage(): any; // we return a placeholder
}

    ```ts
    import * as fs from 'node:fs';
    import * as path from 'node:path';
    import {
      AtlasAttachmentLoader,
      SkeletonJson,
      TextureAtlas,
      Texture,
      TextureFilter,
      TextureWrap,
    } from '@esotericsoftware/spine-core';
    import type { LoadResult, LoaderOptions, SourceDims } from './types.js';
    import {
      AtlasNotFoundError,
      AtlasParseError,
      SkeletonJsonNotFoundError,
    } from './errors.js';

    /**
     * Stub Texture — fabricates a headless Texture instance from atlas metadata.
     * Never reads pixel data. Used by createStubTextureLoader().
     *
     * REQUIRED by F1.3 + N2.3: the math phase must not decode PNGs.
     */
    class StubTexture extends Texture {
      constructor(image: unknown = null) {
        super(image as never);
      }
      setFilters(_min: TextureFilter, _mag: TextureFilter): void { /* no-op */ }
      setWraps(_u: TextureWrap, _v: TextureWrap): void { /* no-op */ }
      dispose(): void { /* no-op */ }
    }

    /**
     * Creates a TextureLoader callback compatible with spine-core's TextureAtlas.
     * The returned function fabricates a StubTexture per page — no file I/O, no decode.
     */
    export function createStubTextureLoader(): (pageName: string) => Texture {
      return (_pageName: string) => new StubTexture(null);
    }

    /**
     * Load a Spine 4.2 skeleton JSON + its sibling atlas headlessly.
     *
     * Contract (per CONTEXT.md):
     *   1. If `skeletonPath` is not readable → SkeletonJsonNotFoundError
     *   2. Else, locate `.atlas` as: opts.atlasPath ?? sibling <same-basename>.atlas
     *   3. If atlas not readable → AtlasNotFoundError
     *   4. Parse atlas with a stub TextureLoader (no PNG decode)
     *   5. Parse skeleton via SkeletonJson(new AtlasAttachmentLoader(atlas))
     *   6. Build sourceDims map: prefer originalWidth/Height ('atlas-orig'), else width/height ('atlas-bounds')
     */
    export function loadSkeleton(skeletonPath: string, opts: LoaderOptions = {}): LoadResult {
      // 1. Read skeleton JSON
      let jsonText: string;
      try {
        jsonText = fs.readFileSync(skeletonPath, 'utf8');
      } catch (cause) {
        throw new SkeletonJsonNotFoundError(skeletonPath);
      }

      // 2. Resolve atlas path
      const atlasPath = opts.atlasPath
        ?? path.join(
             path.dirname(skeletonPath),
             path.basename(skeletonPath, path.extname(skeletonPath)) + '.atlas',
           );

      // 3. Read atlas text
      let atlasText: string;
      try {
        atlasText = fs.readFileSync(atlasPath, 'utf8');
      } catch (cause) {
        throw new AtlasNotFoundError(atlasPath, skeletonPath);
      }

      // 4. Parse atlas (stub textures — no PNG decode)
      let atlas: TextureAtlas;
      try {
        atlas = new TextureAtlas(atlasText);
        const loader = createStubTextureLoader();
        for (const page of atlas.pages) {
          page.setTexture(loader(page.name));
        }
      } catch (cause) {
        throw new AtlasParseError(atlasPath, cause);
      }

      // 5. Parse skeleton
      const attachmentLoader = new AtlasAttachmentLoader(atlas);
      const skeletonJson = new SkeletonJson(attachmentLoader);
      const skeletonData = skeletonJson.readSkeletonData(JSON.parse(jsonText));

      // 6. Build sourceDims
      const sourceDims = new Map<string, SourceDims>();
      for (const region of atlas.regions) {
        const hasOrig = region.originalWidth > 0 && region.originalHeight > 0;
        sourceDims.set(region.name, {
          w: hasOrig ? region.originalWidth : region.width,
          h: hasOrig ? region.originalHeight : region.height,
          source: hasOrig ? 'atlas-orig' : 'atlas-bounds',
        });
      }

      return {
        skeletonPath: path.resolve(skeletonPath),
        atlasPath: path.resolve(atlasPath),
        skeletonData,
        atlas,
        sourceDims,
      };
    }
    ```

    Notes:
    - If `page.setTexture` isn't the exact spine-core 4.2 public API name, use the equivalent (`page.texture = loader(page.name)`). Verify against the spine-core types; don't guess.
    - For `AtlasAttachmentLoader` to work with stub textures, regions must be resolvable by name. The atlas parser already populates `atlas.regions` — no further setup needed.
    - If the `TextureAtlas` constructor signature in 4.2 differs (e.g. requires a textureLoader callback directly), adapt but PRESERVE the contract: stub textures, no PNG decode. Record any adaptation in the SUMMARY.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI argv → `skeletonPath` | User-supplied path is passed to `fs.readFileSync`. Unvalidated input crosses the trust boundary into the filesystem API. |
| `skeletonPath` → `atlasPath` derivation | `path.join(dirname, basename + '.atlas')` resolves relative to the JSON's parent — a symlinked or crafted JSON path could redirect atlas resolution. |
| atlas text → `TextureAtlas` parser | User-controllable string parsed by spine-core's atlas grammar. |
| skeleton JSON bytes → `JSON.parse` → `SkeletonJson.readSkeletonData` | User-supplied JSON parsed by V8's parser, then consumed by spine-core. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-00-02-01 | T (Tampering) / I (Info disclosure) | `fs.readFileSync(skeletonPath)` | accept | CLI is a local dev tool run by the user on their own files. Path traversal has no privilege boundary to cross — the user already has read access to everything under their uid. We do NOT add path sanitization. |
| T-00-02-02 | T (Tampering) | `JSON.parse(jsonText)` + prototype pollution via `__proto__` keys | mitigate | V8's `JSON.parse` does NOT assign `__proto__` as a property (ES2019+ behavior). `SkeletonJson.readSkeletonData` is a structural walker — it only reads specific named properties into typed `SkeletonData` classes, not `Object.assign`-style merging. Mitigation: trust the two-layer defense; do NOT use `Object.assign` or `{...parsed}` anywhere in the loader. Grep-verified in tests: `! grep -q "Object.assign" src/core/loader.ts`. |
| T-00-02-03 | D (DoS) | malicious huge JSON | accept | `fs.readFileSync` is synchronous — a 2 GB JSON would OOM the CLI process. The CLI is per-invocation, user-local; acceptable failure mode. Not mitigated in Phase 0. |
| T-00-02-04 | T (Tampering) | stub TextureLoader side effects | mitigate | `StubTexture` implementations are no-ops; `createStubTextureLoader` does not import `fs`/`child_process`/`net`. Grep-verified: `! grep -qE "(child_process|net|http|dgram)" src/core/loader.ts`. |
</threat_model>
