The user must either (a) Dismiss the existing error banner / wait for the toast to clear, or (b) drop a clean/working .json/.stmproj first to clear the error state, before Cmd+O becomes responsive again.

Adjacent to VR-02's domain (toolbar Open + Cmd+O) but NOT a regression of the VR-02 fix itself — VR-02 specifically tested the data-correctness path (threaded `projectPath` instead of empty string), which is GREEN.
