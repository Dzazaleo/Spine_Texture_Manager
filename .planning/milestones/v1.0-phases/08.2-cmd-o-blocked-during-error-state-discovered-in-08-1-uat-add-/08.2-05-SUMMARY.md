## Patterns lifted

- **Renderer test discipline (React 19 concurrent rendering)** — Plan 8.1-04 / 8.1-05 React 19 fixes carried forward verbatim:
  - `await screen.findByText(/^CIRCLE$/i)` (NOT `await Promise.resolve()`) — testing-library's `waitFor` polls for state-driven UI mutations under React 19's MessageChannel/setTimeout scheduler.
  - CIRCLE-not-TRIANGLE row targeting (makeSummary peaks contain CIRCLE + SQUARE only).
  - `userEvent.dblClick` on the Scale cell of the matched row (the Scale cell carries the `onDoubleClick` handler, NOT the attachmentName cell).
- **`.mock.calls[0][0]` capture pattern** — App.tsx's menu-event `useEffect` calls `window.api.onMenuOpen(cb)` once on mount; tests retrieve the registered callback via `.mock.calls[0][0]` and invoke directly. No round-trip through real Electron needed — this is the correct test boundary for the renderer-side wiring.
- **Per-test mock override** — 8.2-MENU-04 overrides `openProjectFromFile` per-test (`vi.fn().mockResolvedValue(...)`) to thread the SkeletonNotFoundOnLoadError envelope; matches the 8.1-VR-01 reproducer pattern at `save-load.spec.tsx:262-301`.
- **Direct AppShell mount for inverse assertion** — 8.2-MENU-06 mounts AppShell directly (NOT App) so the App.tsx-level menu-event `useEffect` never runs. Even with the defensive `openProject.mockClear()`, the test boundary stays clean: a Cmd+O keydown via `window.dispatchEvent` must NOT cause `openProject` to be called by ANY path within the AppShell render tree.
