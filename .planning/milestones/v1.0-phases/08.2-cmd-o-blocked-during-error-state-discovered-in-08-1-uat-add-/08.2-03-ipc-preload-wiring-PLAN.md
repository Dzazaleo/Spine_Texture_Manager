## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Renderer process → main process via 'menu:notify-state' | Renderer is UNTRUSTED. Main MUST validate the payload field-by-field before mutating `currentMenuState` or invoking Menu.setApplicationMenu — same discipline as `validateExportPlan` (ipc.ts:106-126). |
| Main process → renderer process via 'menu:*-clicked' | One-way `webContents.send`. Main is the trust source; renderer subscribes via Pitfall 9 wrapped const. The `path` argument carried by 'menu:open-recent-clicked' originates from recent.json (validated in Plan 01) → menu builder (Plan 02) → here. |
| Preload `notifyMenuState` argument → IPC payload | Preload trusts its renderer caller (App.tsx + AppShell.tsx); the trust boundary is enforced AT THE MAIN BOUNDARY (`ipcMain.on`'s validation gate). Preload-side validation would be duplicated effort. |
