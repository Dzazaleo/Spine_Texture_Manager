## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-text gap inside tab buttons (`gap-1`) |
| sm | 8px | Compact element spacing, badge margin (`ml-1.5 = 6px` — rounded exception) |
| md | 16px | Default element spacing |
| lg | 24px | Panel horizontal padding (`px-6`) |
| xl | 32px | Panel padding (`p-8`) |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:

- Pill badge left margin: `ml-1.5` (6px) — approved exception for compact tab-strip fit.
  Source: 26.2-CONTEXT.md D-04 and `<specifics>` block.
- Pill badge padding: `px-1.5 py-0.5` (6px / 2px) — compact badge geometry; approved exception.
- Tab button active underline: `h-[2px]` — intentional sub-4px accent line (existing pattern).
- `min-w-[6rem]` counter cell — existing Phase 26.1 D-09 exception, not changed here.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body / table rows | 14px (text-sm) | 400 (font-normal) | 1.5 |
| Label / tab buttons | 14px (text-sm) | 400 inactive / 500 medium active | 1.5 |
| Code / filenames | 14px (font-mono text-sm) | 400 | 1.5 |
| Caption / metadata | 12px (text-xs) | 400 | 1.5 |
| Badge count | 12px (text-xs) | 500 (font-medium) | n/a (single line) |

Declared weights: 2 total — `400 (font-normal)` for body/inactive labels/code/captions,
and `500 (font-medium)` for active tab labels and badge count. The active tab label uses
`font-medium` (500) rather than `font-semibold` (600) to stay within the 2-weight limit;
the badge uses `font-medium` (500) per 26.2-CONTEXT.md D-04 (locked). Both share the
same weight value, keeping the declared set to exactly {400, 500}.

Source: existing codebase tokens, 26.1-CONTEXT.md baseline, 26.2-CONTEXT.md D-04.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--color-surface` = `#232732` | App background, drop zone |
| Secondary (30%) | `--color-panel` = `#232a3e` | Panels, table headers, card backgrounds |
| Accent (10%) | `--color-accent` = `#f97316` (orange-500) | Active tab underline, active tab label, hover states, focus ring |
| Danger | `--color-danger` = `#e06b55` | Pill badge background modifier (`bg-danger/20`), badge text (`text-danger`), danger-themed panel header elements, missing-row icons |

Accent reserved for: active tab underline indicator (`bg-accent`), active tab label text
(`text-accent`), button hover borders (`hover:border-accent`), focus-visible outlines
(`focus-visible:outline-accent`).

Danger reserved for: Unused tab pill badge (`bg-danger/20 text-danger`), panel header
backgrounds for MissingAttachmentsPanel + UnusedAssetsPanel (`bg-danger/10
border-danger/40`), missing-row warning icons (`text-danger`), danger-border accents
(`border-danger/40`).

All color tokens must be used by semantic name only. Raw hex values are forbidden in
JSX className strings. Tailwind v4 literal-class discipline applies: no template
interpolation in className (e.g., `bg-${color}/20` is forbidden).

Source: `src/renderer/src/index.css:44-84`, 26.1-CONTEXT.md D-01..D-03 (color locked),
26.2-CONTEXT.md D-04.

## Icon Contract

This phase establishes and enforces the stroke/outline icon standard across all affected
surfaces. This section is the primary design contract for Phase 26.2.

### Canonical Stroke Icon Standard (D-05)

Every icon in the application now follows this pattern:

```
stroke="currentColor"
strokeWidth="1.5"
strokeLinecap="round"
strokeLinejoin="round"
fill="none"
viewBox="0 0 20 20"
className="w-5 h-5"
aria-hidden="true"
```

Exception: Tab strip icons use `className="w-4 h-4"` (smaller target, compact strip).
Exception: Contextual row icons (missing-row warning triangles) retain `w-4 h-4` to
avoid layout shift inside table cells.

### Tab Strip Icons (D-06)

Each tab button receives a leading inline SVG icon at `w-4 h-4`. Icons use the
canonical stroke style with `viewBox="0 0 20 20"`. Placement: icon sits left of the
label text, separated by `gap-1` (4px).

| Tab | Visual Intent | SVG Path Guidance |
|-----|--------------|-------------------|
| Global | Grid/table — matches GlobalMaxRenderPanel section header | 3×3 grid or table-cells path |
| Unused | Warning triangle — danger-tinted, reinforces problem zone | Equilateral triangle outline with exclamation mark, stroke-only |
| Animation Breakdown | Chart/bars — matches AnimationBreakdownPanel section header | Three vertical bars of ascending height |

The Unused tab icon color: `text-danger` when orphanedFiles.length > 0 (inherits from
pill badge danger context); falls back to `text-fg-muted` when the tab would be hidden
(moot — tab is removed entirely when empty per D-01). In normal rendering, the icon
color matches the tab's active/inactive text color (currentColor).

SVG path data selection is planner's discretion (D-06a). Paths must be visually legible
at `w-4 h-4` (16×16px rendered). Prefer paths with clean geometric geometry (no
micro-detail). If existing inline SVG paths from the Phase 26.1 section-header icons are
reusable at this size, reuse them directly.

### TabButton Component — New Props (D-04, D-06)

The existing `TabButton` function at `AppShell.tsx:1741` gains two new optional props:

```ts
badge?: number       // danger pill count; shown when value > 0
icon?: ReactNode     // leading inline SVG element
```

Rendered layout inside the button: `[icon] [label text] [pill badge]`

Pill badge spec (literal class strings, no interpolation):

```
bg-danger/20 text-danger rounded-full text-xs font-medium px-1.5 py-0.5 ml-1.5
```

Badge is hidden when `badge === 0` or `badge === undefined`. Use a clsx conditional
branch with literal class values — not a template string.

The existing active-underline indicator (`absolute left-0 right-0 -bottom-px h-[2px]
bg-accent`) is unchanged.

### Icons to Convert from Fill to Stroke (D-07)

| File | Line | Current | Target |
|------|------|---------|--------|
| `MissingAttachmentsPanel.tsx` | 65 | `viewBox="0 0 16 16" fill="currentColor"` solid triangle | Stroke triangle, `viewBox="0 0 20 20"`, `w-4 h-4`, `fill="none"`, `stroke="currentColor"` `strokeWidth="1.5"` |
| `UnusedAssetsPanel.tsx` | 59 | `viewBox="0 0 16 16" fill="currentColor"` solid triangle | Same conversion as above |
| `GlobalMaxRenderPanel.tsx` | 461 | `viewBox="0 0 16 16" fill="currentColor"` solid triangle (missing-row ⚠) | Same conversion; retain `w-4 h-4`; no size change (table cell constraint) |
| `AnimationBreakdownPanel.tsx` | 703 | `viewBox="0 0 16 16" fill="currentColor"` solid triangle (missing-row ⚠) | Same conversion; retain `w-4 h-4`; no size change |

Size rule (D-07a): All four converted icons retain `w-4 h-4`. Do not upsize to
`w-5 h-5` — all four sites are inside space-constrained contexts (panel alert banners
and table cells) where upsizing would cause layout shifts.

The converted stroke triangle path must be a proper equilateral triangle outline with
a vertical stroke exclamation mark inside — not a filled triangle with white `<rect>`
cutouts (the current fill approach). The stroke version is visually cleaner and matches
the Phase 26.1 D-12 section-header icon style established for
GlobalMaxRenderPanel/AnimationBreakdownPanel.

## Tab System Contract

### ActiveTab Type (D-01..D-03)

```ts
type ActiveTab = 'global' | 'unused' | 'animation';
```

Default: `'global'`. Change site: `AppShell.tsx:78` (type declaration) and `:162`
(useState initial value — unchanged, still `'global'`).

### Tab Strip Layout

When `orphanedFiles.length > 0`:
```
[ ⊞ Global ] [ ⚠ Unused  12 ] [ ▦ Animation Breakdown ]
```

When `orphanedFiles.length === 0`:
```
[ ⊞ Global ] [ ▦ Animation Breakdown ]
```

The Unused tab button is removed entirely from the DOM when `orphanedFiles.length ===
0` (D-01). No disabled/grayed state — the button simply does not render. Implementation:
conditional JSX `{orphanedFiles.length > 0 && <TabButton …>}`.

### Auto-redirect Guard (D-02)

A `useEffect` watching `[orphanedFiles.length, activeTab]` fires `setActiveTab('global')`
when `activeTab === 'unused' && orphanedFiles.length === 0`. This prevents a stranded
"Unused" active tab state when a resample clears the orphaned files list. Implementation
mirrors the existing pattern for project-change tab reset at `AppShell.tsx:403` and `:416`.

### Panel Rendering Changes

| Panel | Before Phase 26.2 | After Phase 26.2 |
|-------|-------------------|------------------|
| `MissingAttachmentsPanel` | Always rendered above tabs | Unchanged — stays above tabs always |
| `GlobalMaxRenderPanel` | `activeTab === 'global'` gated | Unchanged |
| `UnusedAssetsPanel` | Always rendered (self-hides via return null) | Moved into `activeTab === 'unused'` gate |
| `AnimationBreakdownPanel` | `activeTab === 'animation'` gated | Unchanged |

### SearchBar / Query State (D-03)

The shared `query` state at `AppShell.tsx:185` now drives `UnusedAssetsPanel` in
addition to `GlobalMaxRenderPanel` and `AnimationBreakdownPanel`. `UnusedAssetsPanel`
gains two props:

```ts
query: string
onQueryChange: (q: string) => void
```

Source pattern: `GlobalMaxRenderPanel.tsx` prop interface (reference implementation).

The internal `useState` for `query` inside `UnusedAssetsPanel.tsx:30` is removed. The
inline `<SearchBar>` inside `UnusedAssetsPanel` (lines 88-93) is removed — the sticky
SearchBar in the toolbar is the sole search surface, per the existing pattern for
Global and Animation tabs.

Query does not clear on tab switch — it persists. This is the existing behavior for
Global ↔ Animation switches and extends naturally to the Unused tab.

The `return null` guard at `UnusedAssetsPanel.tsx:47` (when `orphanedFiles.length ===
0`) may be retained as a defensive safety guard or removed since the tab is hidden when
empty. Planner to confirm which is cleaner — either is acceptable.

## Accessibility Contract

- `TabButton` retains `role="tab"` and `aria-selected={isActive}` on all three tab
  buttons (existing pattern, unchanged).
- The `<nav role="tablist">` wrapper at `AppShell.tsx:1327` is unchanged.
- New pill badge: add `aria-label="N orphaned files"` or wrap count in a `<span
  aria-live="polite">` to announce count changes. Recommended: static `aria-label` on
  the badge span is sufficient since count changes only on resample (which reloads the
  whole UI).
- All new SVG icons: `aria-hidden="true"` required. The surrounding element provides
  the accessible name (tab button text, aria-label on containing element).
- Converted fill-to-stroke icons retain their existing `aria-label="Missing PNG"` on the
  wrapping `<span>` — no ARIA changes needed on icon conversion.
- Focus ring: `focus-visible:outline-2 focus-visible:outline-accent` on TabButton
  is unchanged and applies to all three tab buttons.

## Tailwind v4 Discipline

All class strings in this phase must comply with the Tailwind v4 literal-class rule.

Approved literal strings for this phase (executor must use these verbatim):

```
bg-danger/20
text-danger
rounded-full
text-xs
font-medium
px-1.5
py-0.5
ml-1.5
w-4
h-4
w-5
h-5
gap-1
flex
items-center
inline-flex
```

Forbidden patterns:

```js
// NEVER — dynamic construction
className={`bg-${color}/20`}
className={`w-${size}`}

// NEVER — ternary into non-literal interpolation
className={condition ? `text-${shade}` : 'text-fg'}
```

Approved pattern for conditional badge visibility:

```jsx
{badge !== undefined && badge > 0 && (
  <span className="bg-danger/20 text-danger rounded-full text-xs font-medium px-1.5 py-0.5 ml-1.5">
    {badge}
  </span>
)}
```

Approved pattern for conditional tab rendering:

```jsx
{orphanedFiles.length > 0 && (
  <TabButton isActive={activeTab === 'unused'} onClick={() => setActiveTab('unused')} badge={orphanedFiles.length} icon={<WarningTriangleIcon />}>
    Unused
  </TabButton>
)}
```
