function TabButton({
  isActive,
  onClick,
  children,
  icon,
  badge,
}: {
  isActive: boolean;
  onClick: () => void;
  children: ReactNode;
  icon?: ReactNode;
  badge?: number;
}) {
  return (
    <button
      role="tab"
      type="button"
      aria-selected={isActive}
      onClick={onClick}
      className={clsx(
        'relative px-4 py-2 text-sm font-sans transition-colors focus:outline-none focus-visible:outline-2 focus-visible:outline-accent inline-flex items-center gap-1',
        isActive ? 'font-medium text-accent' : 'font-normal text-fg-muted hover:text-fg',
      )}
    >
      {icon}
      {children}
      {badge !== undefined && badge > 0 && (
        <span
          className="bg-danger/20 text-danger rounded-full text-xs font-medium px-1.5 py-0.5 ml-1.5"
          aria-label={badge + ' orphaned files'}

          {badge}
        </span>
      )}
      {isActive && (
        <span
          aria-hidden
          className="absolute left-0 right-0 -bottom-px h-[2px] bg-accent"
        />
      )}

  );

- Badge conditional uses the approved literal pattern from UI-SPEC (no template interpolation).
- All badge class strings are verbatim literals: `"bg-danger/20 text-danger rounded-full text-xs font-medium px-1.5 py-0.5 ml-1.5"`.

**Change 4 — Auto-redirect useEffect when Unused tab disappears (D-02)**

Analog: existing `useEffect(() => { setLocalSummary(null); }, [summary]);` at `AppShell.tsx:245` and the `setActiveTab('global')` calls at lines 403 and 416.

New effect to add (near the other tab-reset guards):
```tsx
// D-02: if the Unused tab disappears (orphaned files cleared by resample),
// redirect back to 'global' to avoid a stranded active-tab state.
useEffect(() => {
  if (activeTab === 'unused' && (effectiveSummary.orphanedFiles ?? []).length === 0) {
    setActiveTab('global');
  }
}, [(effectiveSummary.orphanedFiles ?? []).length, activeTab]);
```

Note: the dependency array watches `orphanedFiles.length` (the primitive) and `activeTab` — mirrors the note in CONTEXT.md D-02.

## Shared Patterns

### Stroke Icon Standard (applies to all 4 icon conversions)

**Source:** UI-SPEC.md §Icon Contract and CONTEXT.md D-05.

The canonical stroke icon target for all converted fill icons:
```tsx
<svg
  viewBox="0 0 20 20"
  fill="none"
  stroke="currentColor"
  strokeWidth="1.5"
  strokeLinecap="round"
  strokeLinejoin="round"
  className="w-4 h-4"   // all 4 sites retain w-4 h-4 (D-07a — no layout shift)
  aria-hidden="true"
>
  {/* warning triangle path + exclamation stroke */}
  <path d="M10 3.5 L17.5 16.5 H2.5 Z" />
  <line x1="10" y1="8" x2="10" y2="12" />
  <circle cx="10" cy="14.5" r="0.5" fill="currentColor" />
</svg>
```

Apply to:
- `MissingAttachmentsPanel.tsx:65`
- `UnusedAssetsPanel.tsx:59`
- `GlobalMaxRenderPanel.tsx:461`
- `AnimationBreakdownPanel.tsx:703`

### Tab Icon Standard (applies to new tab strip icons)

Tab strip icons use the same stroke style but at `w-4 h-4` and are passed as `icon` prop to `TabButton`. They are inline SVGs defined near the TabButton call sites or as small named functions at the bottom of `AppShell.tsx`.
