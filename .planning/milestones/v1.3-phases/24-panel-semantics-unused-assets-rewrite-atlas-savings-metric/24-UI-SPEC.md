## Spacing Scale

Declared values (multiples of 4; existing Tailwind v4 arbitrary-value discipline):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-text gaps (`gap-1`), inline SVG margins (`ml-1`) |
| sm | 8px | Compact padding within chips/badges (`px-2 py-1`, `gap-2`) |
| md | 16px | Panel body padding (`px-4`, `py-4`), row cell padding (`px-3 py-2`) |
| lg | 24px | Panel outer padding (`px-6 py-4`), section bottom margin (`mb-6`) |
| xl | 32px | Panel bottom margin between stacked panels (`mb-4` = 16px, `mb-6` = 24px) |
| 2xl | 48px | Not used in Phase 24 surface area |
| 3xl | 64px | Not used in Phase 24 surface area |

Exceptions:
- Panel border-radius: `rounded-md` (6px) — consistent with all existing panels.
- Table cell padding: `py-2 px-3` — inherited legacy value; verbatim copy of existing GlobalMaxRenderPanel table row padding (`px-3` = 12px, a multiple of 4 but not in the standard set). Confirmed present on 15+ `<td>`/`<th>` elements in `GlobalMaxRenderPanel.tsx` (lines 302, 431, 444, 447–448, 477, 491, 507, 523, 887–889, 899–901, 939, 1049). Changing to `px-2` or `px-4` would diverge from the established table visual rhythm.

Source: existing GlobalMaxRenderPanel, MissingAttachmentsPanel patterns.

### savingsPct chip color rationale
The savings % chip uses `text-warning` (#C9913C) rather than `text-accent` to signal a metric distinct from interactive elements. It is informational, not actionable. WCAG AA: 6.33:1 on `bg-panel` — passes.

## Component Inventory

### New component: UnusedAssetsPanel

Structural analog: `MissingAttachmentsPanel.tsx` (collapsed-by-default banner with expand toggle + list).

**Behavior:**
- Returns `null` when `orphanedFiles.length === 0` — panel is not rendered (no empty-state placeholder). Source: 24-CONTEXT.md D-06.
- Renders expanded by default when `orphanedFiles.length > 0`. Source: 24-CONTEXT.md D-06.
- Panel position in render tree: after `<GlobalMaxRenderPanel>`, before `<AnimationBreakdownPanel>`. Source: 24-CONTEXT.md D-07.

**Header layout:** horizontal flex row containing:
1. Left-edge danger strip (1px wide, full height `h-4`) — same as MissingAttachmentsPanel `w-1 h-4 bg-danger`.
2. Count + bytes callout: `"N orphaned file(s) · X.X KB"` — filename count + `formatBytes(totalBytesOnDisk)` joined by ` · `.
3. Expand/collapse toggle button (right-aligned): `"Show details"` / `"Hide details"`.

**Body (expanded):** table with 2 columns:
- `Filename` — basename, no `.png` extension, `font-mono text-sm text-fg`.
- `Size on Disk` — `formatBytes(entry.bytesOnDisk)`, `font-mono text-sm text-fg-muted`.
- Search filter: `<SearchBar>` component (substring on filename; existing component, reused verbatim).
- Empty filter state: single row `<td colSpan={2}>` with `"(no matches)"` in `text-fg-muted font-mono text-sm text-center py-4`.

**ARIA:** `role="alert"` on outer container (same as MissingAttachmentsPanel; surfaces to screen readers when panel appears). `aria-label="Orphaned image files"`.

### Modified component: GlobalMaxRenderPanel

**Removed:** Entire unused-attachment section (lines 862–907 of current file). `unusedNameSet`, `aggregateBytes`, `filteredUnused` memos removed. `unusedAttachments` local variable removed.

**Added — savingsPct chip in section header:**
- Location: GlobalMaxRenderPanel section `<header>` — right side, after `"N selected / M total"` counter.
- Renders only when `savingsPct !== null && savingsPct > 0`. Hidden when null (no data) or 0.
- Visual: `<span>` with `text-warning font-mono text-xs` containing `"{savingsPct.toFixed(1)}% pixel savings"`.
- Chip framing: `border border-border rounded-md px-2 py-1` (8px horizontal / 4px vertical — standard sm token).
- Prop: `savingsPct?: number | null` added to `GlobalMaxRenderPanelProps`.

**Header final layout (left to right):**
1. Panel icon SVG (existing, unchanged)
2. "Global Max Render Scale" label (existing, unchanged)
3. `ml-auto` spacer
4. savingsPct chip (new, conditional) — `text-warning font-mono text-xs border border-border rounded-md px-2 py-1`
5. `"N selected / M total"` counter (existing, `text-fg-muted font-mono text-sm font-normal`)

**Removed from prop surface:** `unusedAttachments?: UnusedAttachment[]` (type deleted upstream).

## Copywriting Contract

| Element | Copy |
|---------|------|
| Panel header (count, singular) | `"1 orphaned file · X.X KB"` |
| Panel header (count, plural) | `"N orphaned files · X.X KB"` |
| Expand toggle (collapsed) | `"Show details"` |
| Expand toggle (expanded) | `"Hide details"` |
| Table column: filename | `"Filename"` |
| Table column: size | `"Size on Disk"` |
| Empty filter state | `"(no matches)"` |
| savingsPct chip label | `"{N.N}% pixel savings"` |
| Search filter placeholder | `"Filter by filename…"` |
| ARIA label on panel | `"Orphaned image files"` |
| AtlasNotFoundError new sentence | `"Tip: if you have a PNG images folder but no .atlas file, enable the "Use Images Folder as Source" toggle in the toolbar and reload."` |

No primary CTA exists in this phase — Phase 24 introduces informational panels with no user-actionable buttons beyond the expand toggle. No destructive actions.

Empty state: panel hidden entirely (returns null) when `orphanedFiles.length === 0`. There is no "no orphaned files — you're clean" message. Absence of the panel IS the empty state.

Error states: none specific to Phase 24. AtlasNotFoundError already has its own rendering path in AppShell — Phase 24 only amends the error message string in `src/core/errors.ts`.

## Interaction Contract

### UnusedAssetsPanel expand/collapse

- Default state on mount: expanded (when panel renders at all).
- Toggle button: `type="button"`, `aria-expanded={expanded}`. Click toggles `expanded` local state.
- Transition: none (no CSS animation) — consistent with MissingAttachmentsPanel.

### Search filter in UnusedAssetsPanel

- `<SearchBar>` component reused verbatim with `placeholder="Filter by filename…"`.
- Predicate: `entry.filename.toLowerCase().includes(query.trim().toLowerCase())`.
- Renders only when `expanded === true` (hidden in collapsed state).
- Clear button and Escape key behavior: inherited from `SearchBar` component unchanged.

### savingsPct chip

- Read-only display element. No hover state, no click handler.
- Renders only when `savingsPct !== null` (non-null guard from AppShell `savingsPctMemo`).
- No tooltip.
