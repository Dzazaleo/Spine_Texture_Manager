<decisions>
## Implementation Decisions

### Color palette (LOCKED 2026-05-04)
- **D-01: `--color-surface` → `#232732`.** App background, DropZone area, and any element using `bg-surface`.
- **D-02: `--color-panel` shifts proportionally lighter in the blue-dark family.** Research phase determines the exact hex. Rule: the lightness delta between surface and panel must roughly match the current warm-stone relationship (panel ≈ surface + ~10–12 brightness units). Candidate: `#2a3148` or similar. Must pass WCAG AA on text-fg against panel.
- **D-03: All other color tokens unchanged.** Orange accent (`--color-accent` = #f97316), danger, warning, success, fg, fg-muted all stay. Only surface + panel shift.
- **D-04: Modal dialogs need no per-file changes.** They already reference `bg-panel` / `bg-surface` tokens — new values flow through automatically once tokens are updated in `index.css`.

### Full-width panels (LOCKED 2026-05-04)
- **D-05: Remove `max-w-6xl mx-auto` from both panel root divs.** `GlobalMaxRenderPanel:801` and `AnimationBreakdownPanel:399`. Keep or reduce the horizontal padding — research will determine if `p-8` becomes `px-6 py-6` or stays `p-8` (just without the max-width cap).

### Missing row full-row fill (LOCKED 2026-05-04)
- **D-06: Add `bg-danger/10` to the `<tr>` element** in both GlobalMaxRenderPanel and AnimationBreakdownPanel for rows where `state === 'missing'`. The name-cell `bg-danger/10 text-danger` class (Phase 25) stays; the row-level `bg-danger/10` reinforces it across all cells. Left-accent (`bg-danger` solid) is unchanged.

### Warning triangle sizing (LOCKED 2026-05-04)
- **D-07: ⚠ icon sized to match DimsBadge circle (`w-4 h-4`).** Options (research will determine): (a) wrap the Unicode `⚠` in a `<span className="inline-block w-4 h-4 text-sm leading-none flex items-center justify-center">`, or (b) replace with an inline SVG triangle at `w-4 h-4`. Both approaches produce a 16×16px footprint matching the DimsBadge info-circle SVG.

### Atlas counter chip — atlas-less mode (LOCKED 2026-05-04)
- **D-08: `loaderMode === 'atlas-less'` → show `{n} images` instead of `1 atlases`.** `n = effectiveSummary.attachments.count − (effectiveSummary.skippedAttachments?.length ?? 0)`. When `loaderMode === 'auto'`, the chip stays `1 atlases` exactly as today. Change is at `AppShell.tsx:1314`.

### Toolbar height harmonization (UI-06, UI-10 — LOCKED 2026-05-04)
- **D-09: Single height token for toolbar elements.** All toolbar buttons use `h-8 px-3` (or `py-1.5 px-3`) — same computed height. SearchBar input gets matching height. Checkbox label row aligns to center. The `N selected / N total` counter cell in the Global panel gets `min-w-[6rem]` to prevent column shift.

### Zebra striping (UI-07 — LOCKED 2026-05-04)
- **D-10: Odd rows transparent, even rows `bg-white/[0.03]`.** Applied via a Tailwind even-row modifier (`even:bg-white/[0.03]`) on `<tr>` in both panels. The existing `rowState`-driven colors (accent, danger, warning) must override zebra — specificity to be confirmed during research. Missing rows (`bg-danger/10` from D-06) take priority over even/odd tinting.

### Danger-themed panel headers (LOCKED 2026-05-04)
- **D-11: MissingAttachmentsPanel and UnusedAssetsPanel headers** adopt `bg-danger/10 border-danger/40` + a danger-colored icon (⚠ or similar) + bold title. Same header height as today (no layout change). Color reinforces "this is a problem zone" before the user opens the section.

### Stronger section headers (LOCKED 2026-05-04)
- **D-12: Global Max Render Scale and Animation Breakdown** panels get an internal header row with: SVG icon (grid/table for Global, chart/breakdown for Animation), bold title, and a right-aligned count chip. The count chip mirrors what the sticky bar shows for each panel. Research determines exact icon set (existing Heroicons or inline SVG).

### Claude's Discretion
- **Exact `--color-panel` value**: research the lightness math and pick the hex that passes WCAG AA and feels visually proportional. Candidate range: `#282f44` – `#2e3650`.
- **Row even/odd specificity**: if Tailwind `even:` modifier conflicts with `rowState` overrides, use a ternary in JSX (`even ? 'bg-white/[0.03]' : ''`) — the existing pattern in GlobalMaxRenderPanel already uses clsx conditionals, so this is natural.
- **Warning triangle approach**: prefer the inline SVG triangle for precision; fall back to the styled `<span>` wrapper if SVG adds excessive complexity.
- **Section header icons**: pick from the existing Heroicons subset already used in the app (TableCellsIcon, ChartBarIcon, or similar from Heroicons outline set at 16px). Do not add a new icon library.

</decisions>

<code_context>
## Existing Code Insights

### Reusable Assets
- `DimsBadge.tsx` — defines the `w-4 h-4` SVG icon pattern; warning triangle should match this
- `clsx` — already imported in all panel files; use for conditional row classes
- Existing `rowState()` predicate in both panels — zebra-stripe condition should be additive (not replace rowState colors)

### Established Patterns
- **Tailwind v4 literal-class discipline**: every className must be a string literal or a clsx conditional with literal branches. No template interpolation. Any new class string must appear verbatim in the source (no dynamic construction like `bg-${color}/10`).
- **Token-based colors**: never use raw hex in className. Always use the CSS custom property tokens (`bg-surface`, `bg-panel`, `text-fg`, etc.). The new `#232732` and panel color live in `index.css` tokens — JSX files use the token names only.
- **`even:` Tailwind modifier**: confirm it works in Tailwind v4 for `<tr>` elements. If not, use the JSX index-based `i % 2 === 0` pattern already present in some virtualized row renderers.

### Integration Points
- `AppShell.tsx:1308-1316` — atlas counter chip reads `loaderMode` (already in scope as state); `effectiveSummary.skippedAttachments` is also available
- Toolbar height: `AppShell.tsx:1337-1417` — all button/input elements in the right-aligned cluster; harmonize `py-1.5` / `h-8` across all
- `src/renderer/src/index.css` — single file for all token changes; update here only
