| renderer → main (saveProject IPC) | Renderer passes AppSessionState containing lastOutDir (a user-picked folder path). Main writes the .stmproj file at currentProjectPath. |

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-23-01 | Tampering | AppShell.tsx:onConfirmStart — lastOutDir stored in React state | accept | lastOutDir originates from OS picker result (window.api.pickOutputDirectory return); it is never parsed from user-typed input. The OS picker is a trusted native boundary. No sanitization needed beyond what the OS already enforces. |
| T-23-02 | Tampering | AppShell.tsx:onRunEnd — silent saveProject writes lastOutDir to .stmproj | mitigate | buildSessionState() reads lastOutDir from React state (set only from OS picker result). main/ipc.ts handleSaveProject validates the ProjectFileV1 schema via serializeProjectFile — malformed state is rejected at the main boundary. Existing schema validation covers lastOutDir: string|null type check. |
| T-23-03 | Information Disclosure | lastOutDir path string in .stmproj and in OptimizeDialog header title | accept | lastOutDir is the user's own chosen output folder. It is displayed in the app's own header (same UX as the existing "N images → /path" display). No escalation beyond what the user already has access to. |
| T-23-04 | Elevation of Privilege | pickOutputDir(startPath) — renderer supplies startPath hint to main | accept | startPath is used only as the dialog's initial directory hint (defaultPath in Electron's showOpenDialog).
