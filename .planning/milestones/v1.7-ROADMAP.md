## Overview

v1.7 turns one full-size Spine export into a fan-out of faithful, genuinely-smaller rig variants — each a complete drop-in package (scaled skeleton JSON + scaled atlas + resized textures, in its own folder) sized to the peak render demand of *that* smaller rig. The risky core was de-risked before planning: spikes 001–003 (all VALIDATED) proved a JSON→JSON transform mirroring spine-core `SkeletonJson.scale` is **field-identical** to Spine's own scaling on 4.2 + 4.3 (including DEMON's worst 4.3 constraints), and that the baked variant is geometrically `s×` for every attachment including constraint-driven `R_ARM`.

The milestone started as four dependency-ordered phases (48–51) and was deliberately kept open by the owner to add two more (52, 53) rather than defer them to v1.8: Phase 48 promotes the spike `baker.mjs` into a Layer-3-pure `core/` bake module, finishes the finite remaining work (constraint-timeline curve channels), and wires the decisive sampling-free oracle as a CI test across a fixture matrix that **includes a deform-heavy rig**. Phase 49 ships the first end-user value (single scale → one folder) and is the first feature to ever make the app **write** a skeleton JSON. Phase 50 adds the setup-pose rig-bounds reference and the two-way scale↔dimension input. Phase 51 fans it out to batch (N scales → N folders). Phase 52 hardens the batch path against partial failure and clears the Phase-51 review backlog. Phase 53 persists the variant scale rows + output location in `.stmproj`. Two hard threads run through every phase: dual-runtime (4.2 + 4.3) and both loader modes (atlas-source + atlas-less); `core/` stays pure-TS (Layer-3, `tests/arch.spec.ts`). The source JSON is never modified.

## Milestone Summary

**Delivered:** A complete multi-scale variant exporter — from one full-size Spine export, the animator produces faithful scaled-down rig variants (scaled skeleton JSON + resized textures + scaled atlas, each in its own folder), single or batch, sized to each smaller rig's own peak render demand, across dual-runtime (4.2 + 4.3) and both loader modes — without ever modifying the source project. Built on a proven Layer-3-pure JSON→JSON bake that is field-identical to Spine's own `SkeletonJson.scale`.

**Key Decisions:**

- Variants are produced by a **full Spine-style similarity bake** (= `SkeletonJson.scale` as a JSON→JSON transform), NOT by scaling a bone (root explodes; pivot leaves constraint residual). Falsified in SEED-010, proven in spikes 001–003.
- `variant_peak = s × master_peak` (arithmetic) — NEVER re-sample the baked variant (the sampler's `peakScale` is invariant under the bake; re-sampling is a measurement blind spot).
- Decisive correctness gate is a sampling-free round-trip oracle `parse(bake(orig,s),1) ≡ parse(orig, SkeletonJson.scale=s)` run in CI across a fixture matrix incl. a deform-heavy rig (DEMON 4.3 has no deform → false confidence alone).
- The variant package reuses `buildExportPlan` + the image-worker + the atlas-writer UNCHANGED; the baked scaled JSON is the one always-present new artifact (textures/atlas follow the chosen `loose|atlas|both` mode).
- Output layout `{PARENT}/{NAME}@{s}x/` with clean inner basenames — designed as the no-collision batch fan-out.
- Single + batch unify in ONE Export Variant dialog (1 row = single, 2+ = batch); single pane, NO tabs. Optimize Assets stays a SEPARATE, byte-untouched flow.
- Batch is continue-on-error: a duplicate/failed scale fails only its own row(s); each folder is atomic with its own rollback; a failed variant leaves no orphan empty folder.
- `.stmproj` persistence is additive-optional (`variantRows`) reusing the existing `lastOutDir` for the output half — no schema version bump (`V_LATEST` stays 1).
