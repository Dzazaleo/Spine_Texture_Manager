# Phase 20 тАФ UI Design Contract

> Visual and interaction contract for the Documentation Builder feature. Generated by gsd-ui-researcher; verified by gsd-ui-checker.
>
> **Two surfaces are governed by this spec:**
> 1. **Renderer modal** тАФ `DocumentationBuilderDialog.tsx` (10th hand-rolled modal). Uses warm-stone Tailwind v4 tokens already locked in `src/renderer/src/index.css`.
> 2. **HTML export** тАФ `src/main/doc-export.ts` template-literal output. Inline `<style>` block with **literal CSS** (NOT Tailwind) per D-17. Self-contained: no external CSS, no fonts, no images, no scripts.
>
> Where the two surfaces diverge, both columns are spelled out. Where they converge (palette, type scale), one row applies to both.

## Spacing Scale

All spacing in BOTH the renderer modal AND the HTML export uses Tailwind's default 4-px grid (multiples of 4 only). Existing modals (OptimizeDialog, OverrideDialog, AtlasPreviewModal) sample exactly the values below тАФ Phase 20 reuses verbatim.

| Token | Tailwind | Pixels | Usage |
|-------|----------|--------|-------|
| xs | `gap-1`, `p-1`, `mb-1` | 4px | Icon-glyph margin from label, LOOP pill `px-1.5` (6px is the lone half-step exception, locked by Phase 19 LOOP pill class) |
| sm | `gap-2`, `p-2`, `px-2` | 8px | Compact inline spacing; chip-strip inter-chip gap; inline-form label-to-input |
| md | `gap-3`, `p-3`, `px-3`, `py-3` | 12px (Tailwind unit-3) | Card body padding; button `px-3 py-1`; tab strip horizontal padding. **Note:** Tailwind unit-3 = 12px, NOT a 4-pt-grid violation (12 = 4 ├Ч 3) |
| md-4 | `gap-4`, `mb-4` | 16px | Default element gap; section-header to body |
| lg | `p-6`, `mb-6` | 24px | Modal outer padding (`p-6` matches all 9 existing modals тАФ `OptimizeDialog.tsx:271`) |
| xl | `gap-8`, `mt-8` | 32px | Major HTML-export section breaks (between Hero row and chip strip; between cards) |
| 2xl | `mt-12`, `gap-12` | 48px | Reserved тАФ none used in this phase |

**Exceptions:**
- `px-1.5 py-0.5` (6px / 2px) on the LOOP pill тАФ locked by Phase 19 inline-pill class string (Claude's Discretion in CONTEXT.md). This is the ONLY half-step exception and is renderer-only; HTML export renders LOOP pills with literal `padding: 2px 6px`.
- `text-[10px]` on the LOOP pill тАФ non-scale font size, also locked by Phase 19.

## Typography

The renderer modal uses Tailwind's default type scale (`text-xs`/`text-sm`/`text-base`/`text-lg`/`text-2xl`); the HTML export uses **literal CSS pixel values** in the inline `<style>` block. Both surfaces share the SAME numeric scale; the HTML export just spells the pixels.

### Renderer modal (Tailwind v4 utility classes)

| Role | Class | Pixels | Weight | Line Height | Usage |
|------|-------|--------|--------|-------------|-------|
| Body | `text-sm` | 14px | `font-normal` (400) | `leading-normal` (1.5) | Default for table cells, descriptions, freeform notes |
| Label | `text-xs` | 12px | `font-semibold` (600) | `leading-normal` (1.5) | InfoCard sub-labels ("Used Files"), tab labels, button labels, chip labels |
| Heading | `text-base` | 16px | `font-semibold` (600) | `leading-snug` (1.375) | Modal title (`<h2 id="documentation-title">`), card headers, pane sub-headings |
| Display | `text-2xl` | 24px | `font-semibold` (600) | `leading-tight` (1.25) | Reserved for hero numerics (matches OptimizeDialog's summary tile pattern). Phase 20 uses it ONLY in the HTML export's "1%" Safety Buffer + "91.7%" Space Savings stat tiles, NOT in the modal |

**Locked weight count: 2** тАФ `font-normal` (400) and `font-semibold` (600). NO `font-medium`, NO `font-bold`. Mirrors every existing modal.

**Locked size count: 4** тАФ 12, 14, 16, 24. The 10px LOOP pill is a brand-pill exception, not a body-type role.

### HTML export (literal CSS)

| Role | Selector / Inline | Pixels | Weight | Line Height | Usage |
|------|--------------------|--------|--------|-------------|-------|
| Body | `body { font-size: 14px; line-height: 1.5; }` | 14px | 400 | 1.5 | Default for muted descriptions, table cells, notes |
| Label | `.chip, .stat-label, th { font-size: 12px; font-weight: 600; }` | 12px | 600 | 1.5 | Chip strip, stat sub-labels, table column headers |
| Heading | `.card-header, h2, .track-divider { font-size: 16px; font-weight: 600; }` | 16px | 600 | 1.375 | Card headers, "тЧП TRACK N" dividers |
| Display | `.stat-value, .hero-skeleton { font-size: 24px; font-weight: 600; }` | 24px | 600 | 1.25 | "1%" Safety Buffer, "91.7%" Space Savings, `<SKELETON_NAME>` in hero row |

**Hero-row title** (`.hero-title`) renders "Spine Documentation /" in 16px white + `<SKELETON_NAME>` in 24px terracotta side-by-side; the trailing slash + name uses the Display size to read as the page identity per the user's screenshot.

## Interaction Contracts

> Locked behaviors. Every contract here has a single canonical source.

### Drag-and-drop (Animation Tracks)

| Event | Contract |
|-------|----------|
| `dragstart` on side-list `<li>` | `e.dataTransfer.setData('application/x-stm-anim', animationName)` AND `e.dataTransfer.effectAllowed = 'copy'` (Electron Chromium quirk per CONTEXT.md D-06) |
| `dragover` on track container | `e.preventDefault()` (mandatory to enable drop) |
| `drop` on track container | Read `e.dataTransfer.getData('application/x-stm-anim')`, append `{ id: crypto.randomUUID(), trackIndex, animationName, mixTime: 0.25, loop: false, notes: '' }` |
| Drag image visual | Browser-default (do NOT call `setDragImage`) per Claude's Discretion |
| Multiple-add behavior | Same animation can appear multiple times on the same track or across tracks; no warning per D-08 |

### Reorder (within a track)

| Event | Contract |
|-------|----------|
| Click тЖС button | Swap entry with previous in same track |
| Click тЖУ button | Swap entry with next in same track |
| тЖС disabled state | First entry in track |
| тЖУ disabled state | Last entry in track |
| Cross-track move | Not supported; user must remove + re-add (D-07) |

### Filter (Control Bones)

| Behavior | Contract |
|----------|----------|
| Debounce | 100ms per Claude's Discretion |
| Match | Case-insensitive substring on bone name |
| Empty match copy | "No bones match \"{QUERY}\"." |
| Performance | Pure renderer-side filter; no IPC |

### Round-trip drift (D-09 events, D-10 bones, D-11 skins)

| Drift case | Behavior |
|-----------|----------|
| Saved entry whose `name` no longer exists in skeleton | Drop silently on reload |
| Skeleton entity present but missing from saved doc | Auto-add with empty description on reload |
| User-defined entries (`controlBones` checked-off only) | Only saved when `description.length > 0` |
