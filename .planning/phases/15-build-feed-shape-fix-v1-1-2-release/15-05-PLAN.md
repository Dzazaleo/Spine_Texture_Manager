**Hotfix v1.1.2 → v1.1.3 macOS auto-update URL-mismatch defect (D-15-LIVE-1).**

Surfaced in live UAT 2026-04-29T19:30Z (Test 7, Leo, macOS): HTTP 404 on Squirrel.Mac swap. Root cause: published `latest-mac.yml` advertises `url: Spine Texture Manager-1.1.2-arm64.zip` (SPACES); GitHub Releases stores the asset as `Spine.Texture.Manager-1.1.2-arm64.zip` (DOTS); electron-updater 6.x sanitizes spaces to DASHES when fetching → `Spine-Texture-Manager-1.1.2-arm64.zip` → 404. Three names, no agreement. UPDFIX-01 IS NOT closed in v1.1.2.

**Remediation chosen — Option A (synthesizer rewrites url to GitHub-canonical form).**

The synthesizer (`scripts/emit-latest-yml.mjs`) gets a `sanitizeAssetUrl()` helper that replaces literal spaces with dots in the emitted `files[].url` and legacy top-level `path:` fields. sha512 + size compute path is unchanged (those are content-intrinsic; GitHub's rename does not modify content). Local filenames in `release/` remain spaced (`Spine Texture Manager-1.1.3-arm64.zip`) — testers' drag-to-Applications UX is preserved; INSTALL.md needs no edit.

**Why Option A over Option B (electron-builder.yml `artifactName` change):**

The Phase 15 brief's Option B claim that `${productFilename}` resolves to `Spine.Texture.Manager` (dots) is **empirically false** — verified locally:

```bash
$ node -e "const sanitize=require('sanitize-filename'); console.log(JSON.stringify(sanitize('Spine Texture Manager')))"
"Spine Texture Manager"
```

`sanitize-filename` strips reserved chars (`/`, `\`, `?`, etc.) but does NOT replace spaces. A correct Option B requires hardcoding the dotted name as a literal string in `artifactName: 'Spine.Texture.Manager-${version}-${arch}.${ext}'` — divorcing artifactName from the `${productName}` template variable. That's fragile to future productName edits, breaks tester muscle memory for INSTALL.md filename references (`Spine-Texture-Manager-<version>-arm64.dmg` placeholder pattern), and changes the .dmg + .zip + .exe + .AppImage names testers see when downloading manually. Surface: electron-builder.yml + INSTALL.md (~6 lines) + tests/integration/emit-latest-yml.spec.ts fixtures (8 string literals) + tests/integration/build-scripts.spec.ts. Option A's blast radius is 1 helper function + 4 test assertions; the local artifact UX is preserved verbatim.

**Companion regression test (load-bearing pre-flight gate):**

A new describe block in `tests/integration/emit-latest-yml.spec.ts` asserts that NO `files[].url` value emitted by the synthesizer contains a literal space — for ALL three platforms. This is the gate that would have caught D-15-LIVE-1 before tag push. The test is part of the existing 3-OS CI matrix (release.yml:34); failure surfaces simultaneously on ubuntu-latest, windows-2022, macos-14.

**Version bump 1.1.2 → 1.1.3:**

 *     - electron-updater req:  `Spine-Texture-Manager-1.1.3-arm64.zip` (DASHES)
 *   The fix: emit the url field in latest-*.yml as the GitHub-stored DOT form.
 *   electron-updater reads the dot form, builds the download URL from it
 *   verbatim (no further sanitization needed because dots are not whitespace),
 *   and the URL agrees byte-for-byte with the GitHub-stored asset name → 200.
 *
 * Why ONLY space→dot (not multi-space-collapse, not URL encode, not NFD):
 *   GitHub's rename is a deterministic 1:1 substitution per character. Two
 *   consecutive spaces in the source filename produce two consecutive dots
 *   in the stored filename (verified by the multi-space negative test in
 *   tests/integration/emit-latest-yml.spec.ts). Any other transformation
 *   (regex \s+ collapse, encodeURIComponent, normalize 'NFD') would diverge
 *   from GitHub's behavior and re-introduce the 3-name mismatch class.
 *
 * sha512 + size compute path is UNCHANGED — those read bytes from the local
 * file (which still has the spaced name); the hash + byte count are intrinsic
 * to file content, not the asset's stored URL.
 *
 * @param {string} localFilename - Filename as it appears in release/ (may contain spaces)
 * @returns {string} - GitHub-canonical form (spaces replaced with dots; all other chars preserved)
 */
function sanitizeAssetUrl(localFilename) {
  return localFilename.replace(/ /g, '.');
}
