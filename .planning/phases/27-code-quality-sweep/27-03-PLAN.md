## CRITICAL DETERMINISM BOUNDARY — atlas-preview-view.ts and export-view.ts are OUT OF SCOPE

This is the most important scope discipline note in this plan. **Read it before any edit.**

The original revision-1 draft of this plan included `src/renderer/src/lib/atlas-preview-view.ts` and `src/renderer/src/lib/export-view.ts` in scope. **That was incorrect** — both files contain `localeCompare` call sites that LOOK like display sorts but are actually **packer / export-plan determinism contracts**:

- `src/renderer/src/lib/atlas-preview-view.ts:83-86` is the deterministic sort that feeds the MaxRectsPacker at lines 96-98. The file's own comment (lines 81-82) says: "Determinism: sort by sourcePath then attachmentName so two runs over the same summary produce byte-identical packer output (matches src/core/export.ts:223)." This is the renderer-side mirror of `src/core/atlas-preview.ts:91-95`, which is the byte-identical core implementation per **D-125** (preview↔export single source of truth, declared in `src/core/atlas-preview.ts:17,39` and the file header at lines 1-43).

Reordering rows here would diverge the renderer's export-plan view from the core's on-disk export output.

**If this plan changed those two files but did NOT change the matching `src/core/*` files (which it correctly cannot, because the core files are deliberately untouched), the result would be:**

- Renderer preview modal: numeric-collation packed layout
- On-disk export (unchanged `src/core/atlas-preview.ts` + `src/core/export.ts`): lexicographic packed layout
- Preview ≠ export → **D-125 broken** → functional behavior change → **Phase 27 goal violated.**

**The boundary:** `localeCompare` sites in `src/renderer/src/lib/{atlas-preview-view,export-view}.ts` are NOT display sorts. They are determinism contracts feeding either a packer or a downstream consumer that compares against `src/core/*` byte-identical mirrors. They are out of scope for QA-03 — same category as `src/core/*` itself.

**The ONLY in-scope file for QA-03 is `src/renderer/src/panels/GlobalMaxRenderPanel.tsx`** at lines 241-264 (`compareRows`). Verified at planning time: the panel's `compareRows` operates on already-built `EnrichedRow` for table display only; its output is consumed exclusively by `sortRows` (line 266) which produces the table render order. Nothing downstream of `compareRows` crosses a determinism boundary — no packer input, no on-disk byte output, no comparison against a core mirror.


`compareRows` is consumed by `sortRows` at lines 266-269:
```typescript
function sortRows(rows: readonly EnrichedRow[], col: SortCol, dir: SortDir): EnrichedRow[] {
  const sorted = rows.slice().sort((a, b) => compareRows(a, b, col));
  if (dir === 'desc') sorted.reverse();
  return sorted;
}
```

`sortRows` produces the array rendered into the table body. Its output never crosses a process boundary, never feeds a packer, and is never compared byte-for-byte against `src/core/*`. It is exclusively a display sort. This is the property that makes QA-03's localeCompare update safe at this site (and the property that does NOT hold for atlas-preview-view.ts / export-view.ts — those output arrays DO feed determinism-bound consumers).

Standard option object (one canonical form, used at every site):
```typescript

```

The `localeCompare` second arg is `locales`; we pass `undefined` to use the runtime default, then the option object as third arg:
