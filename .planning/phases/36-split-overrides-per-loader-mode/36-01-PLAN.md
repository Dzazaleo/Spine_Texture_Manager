## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-36-01 | Tampering | `validateProjectFile` validator — hostile `overridesAtlasLess` payload (non-object, array, null, prototype-pollution keys like `__proto__`, `constructor`) | mitigate | Three-layer guard mirrors the existing `overrides` validation pattern (lines 136-141 + 264-278): (1) `obj.overridesAtlasLess === undefined` → substitute `{}`; (2) `!obj.overridesAtlasLess || typeof !== 'object' || Array.isArray(...)` → return `invalid-shape` error; (3) per-key `typeof v !== 'number' || !Number.isFinite(v)` → return `invalid-shape` error. The `Object.entries` iteration ignores prototype-chain keys (only own enumerable keys), so `__proto__` and `constructor` poisoning vectors are nullified. Same guarantees the existing `overrides` field has had since v1.0. |
| T-36-02 | Information disclosure | Field rename leak — readers of the OLD `mergedOverrides` field path in `SerializableError` would silently see `undefined` (data loss for the inactive bucket on locate-skeleton recovery) if the rename is incomplete | accept (in this plan) | This plan only renames the type; the consumers in `src/main/project-io.ts` (Plan 36-02) and `src/renderer/src/components/AppShell.tsx` + `src/renderer/src/App.tsx` (Plan 36-03) are updated atomically. The phase quality gate (sub-task in Plan 36-05) re-runs `grep -r 'mergedOverrides[^B]' src/ | grep -v '\.test\.'` to confirm zero stale readers remain. Severity: low (no PII; project-local files). Rename caught at compile-time across all consumers via TypeScript. |
| T-36-03 | DoS / resource | Very large `overridesAtlasLess` payload (e.g. 1e6 keys) | accept | No DoS surface — the validator's `Object.entries` walk is O(n) on a user-owned local file. The same bound has always existed for `overrides`. No new attack surface. |

ASVS L1 alignment: input validation at trust boundary (V5.1.x); per-key type validation enforced. No `high` severity threats — security gate not triggered.
