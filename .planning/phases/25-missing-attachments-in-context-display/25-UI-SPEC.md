### 1. `RowState` type extension (both panels)

**Current:**
```ts
type RowState = 'under' | 'over' | 'unused' | 'neutral';
```

**After Phase 25:**
```ts
type RowState = 'under' | 'over' | 'unused' | 'neutral' | 'missing';
```

The `'missing'` variant is checked FIRST in `rowState()`, before `isUnused`, before
ratio comparisons. A missing row is unambiguously dangerous regardless of its stub scale data.

**New `rowState` signature (both panels):**
```ts
function rowState(peakRatio: number, isUnused: boolean, isMissing?: boolean): RowState {
  if (isMissing) return 'missing';
  if (isUnused) return 'unused';
  if (peakRatio < 1.0) return 'under';
  if (peakRatio > 1.0) return 'over';
  return 'neutral';
}
```

### 2. Left-accent bar — both `GlobalMaxRenderPanel` and `AnimationBreakdownPanel`

The existing `<td className="w-1 p-0">` / `<span className="inline-block w-1 h-full ...">` pattern
gains one additional clsx branch:

```tsx
state === 'missing' && 'bg-danger',
```

Placed alongside the existing `state === 'unused' && 'bg-danger'` line.
Visual result: a 4px-wide solid `#e06b55` left stripe spanning the full row height.

**Left-accent bar clsx block after Phase 25 (both panels):**
```tsx
<span
  className={clsx(
    'inline-block w-1 h-full',
    state === 'under' && 'bg-success',
    state === 'over' && 'bg-warning',
    state === 'unused' && 'bg-danger',
    state === 'missing' && 'bg-danger',
    state === 'neutral' && 'bg-transparent',
  )}
  aria-hidden="true"
/>
```

### 3. Attachment name cell — both panels

**Modification site:**
- `GlobalMaxRenderPanel.tsx:452` — the `<td>` containing `{highlightMatch(row.attachmentName, query)}`
- `AnimationBreakdownPanel.tsx:671` — the `<td>` containing `{highlightMatch(row.attachmentName, query)}`

**Current:**
```tsx
<td className="py-2 px-3 font-mono text-sm text-fg">
  {highlightMatch(row.attachmentName, query)}
</td>
```

**After Phase 25:**
```tsx
<td className="py-2 px-3 font-mono text-sm text-fg">
  {row.isMissing && (
    <span className="text-danger mr-1" aria-label="Missing PNG">
      ⚠
    </span>
  )}
  {highlightMatch(row.attachmentName, query)}
</td>
```

Icon placement: BEFORE the attachment name text.
Icon spacing: `mr-1` (4px right margin) between ⚠ and name text.
Icon aria: `aria-label="Missing PNG"` on the wrapping `<span>` — the character itself
conveys the signal to screen readers via the label. The character is NOT `aria-hidden`.
Icon size: inherits `text-sm` from the parent `<td>` — no separate size override needed.

### 4. Tinted ratio cell — both panels

The Scale ratio `<td>` gains a `missing` branch alongside the existing state branches:

```tsx
state === 'missing' && 'bg-danger/10 text-danger',
```

This matches the `unused` treatment exactly. Visual: 10% opacity danger background with
danger-colored scale value text. The `0.000×` stub scale value renders normally per D-02
(all cells render as-is; visual signal carries the "unreliable data" meaning).

## `rowState` Call-Site Updates

Both panels call `rowState()` inside the row render path (GlobalMaxRenderPanel via
`BreakdownRowItem` props pre-computed in parent; AnimationBreakdownPanel similarly).

The call sites must thread `row.isMissing` through as the new third argument.

**GlobalMaxRenderPanel pattern (pre-compute in the row map / useMemo):**
```ts
const state = rowState(row.effectiveScale, isUnused, row.isMissing);
```

**AnimationBreakdownPanel pattern (same):**
```ts
const state = rowState(row.effectiveScale, /* isUnused= */ false, row.isMissing);
```

Note: AnimationBreakdownPanel comment in the existing code acknowledges `isUnused` is
always `false` for breakdown rows (unused state is global-level only). The `isMissing`
flag has no such restriction — a stub row appears in both the global peaks AND per-animation
breakdown cards, so `isMissing` is meaningfully `true` in both contexts.

## `summary.ts` Data Contract (main process)

Not a renderer concern, but included here for completeness so the planner can allocate

Phase 25 removes two filters and adds `isMissing` marking at two sites:

Remove: `peaksArray = peaksArrayRaw.filter((p) => !skippedNames.has(p.attachmentName))`
Replace with: `peaksArray = peaksArrayRaw.map((p) => ({ ...p, isMissing: skippedNames.has(p.attachmentName) ? true : undefined }))`

**Site 2 — breakdown card rows (line 114):**

already available at both sites — no new computation required.

## Visual Consistency Reference

The `MissingAttachmentsPanel` (Phase 21 Plan 21-10) established the danger-accent idiom:
- `bg-danger` left stripe: `<span className="inline-block w-1 h-4 bg-danger flex-shrink-0" />`
- `text-danger` text: `<span className="font-semibold text-danger">{count} attachment{plural} missing PNG{plural}</span>`
- Plain Unicode `⚠` not used in the panel itself (uses count text), but the `text-danger`
  convention for danger signals is established.

Phase 25 extends this idiom into row-level granularity:
- Left stripe width stays `w-1` (4px) — same as panel header stripe AND existing row state bar.
- `⚠` character in `text-danger` at `text-sm` matches row text scale.

Token note preserved from Plan 21-10 ISSUE-010: the project has no `border-warning` token,
so this phase continues using `bg-danger` / `text-danger` for missing attachment signals
(which are warnings, not errors). A future polish phase may add `border-warning` if the
semantic distinction matters.
