## Tailwind v4 Discipline (mandatory for executor)

Every `className` string must be a **string literal** or a `clsx` conditional
with **literal branches**. No template interpolation (e.g., no `bg-${color}/10`).
Every token reference must use the CSS custom property name (e.g., `bg-surface`,
`bg-danger/10`) — never raw hex in JSX. New class strings must appear verbatim
in source so the Tailwind v4 scanner can include them.

Source: CONTEXT.md `<code_context>` Tailwind v4 literal-class discipline.

## Spacing Scale

Standard 8-point scale (multiples of 4). No new spacing tokens introduced in
this phase — all changes reuse existing Tailwind scale values.

| Token | Value | Usage in this phase                              |
|-------|-------|--------------------------------------------------|
| xs    | 4px   | Icon gaps (`gap-1`), inline micro-spacing        |
| sm    | 8px   | Compact element spacing (`gap-2`, `p-2`)         |
| md    | 16px  | Default element spacing (`px-3` ≈ 12px; `p-4`)  |
| lg    | 24px  | Section padding (`px-6`, `py-6`)                 |
| xl    | 32px  | Panel content padding (`p-8` where retained)     |
| 2xl   | 48px  | (no change in this phase)                        |
| 3xl   | 64px  | (no change in this phase)                        |

**Exceptions:**

- Toolbar buttons: `h-8` (32px computed height) with `px-3` horizontal padding.
  `h-8` is the single height token for all toolbar elements (D-09). This is a
  deliberate exception to the default `py-1` button sizing to achieve uniform
  toolbar height.
- `N selected / N total` counter cell: `min-w-[6rem]` (96px) prevents
  layout-shift on count changes (D-09).
- Panel root divs: horizontal padding stays `p-8` (32px all sides) after
  removing `max-w-6xl mx-auto` — executor may reduce to `px-6 py-6` if visual
  testing shows excessive whitespace, but must not re-introduce the max-width
  constraint (D-05).

Source: CONTEXT.md D-05, D-09.

## Color

### Token Changes (D-01–D-04)

The ONLY file to edit for color tokens is `src/renderer/src/index.css`.
All JSX files consume tokens by name — no JSX changes for color tokens.

| Token              | Old value (warm stone)       | New value (cool blue-dark)  | Source |
|--------------------|------------------------------|-----------------------------|--------|
| `--color-surface`  | `var(--color-stone-950)` → `#0c0a09` | **`#232732`** | D-01 LOCKED |
| `--color-panel`    | `var(--color-stone-900)` → `#1c1917` | **`#2c3347`** | D-02 LOCKED (see math below) |
| `--color-border`   | `var(--color-stone-800)` → `#292524` | **`#384158`** | Proportional shift (see math below) |
| `--color-fg`       | `var(--color-stone-100)` → `#f5f5f4` | unchanged                   | D-03 |
| `--color-fg-muted` | `var(--color-stone-400)` → `#a8a29e` | unchanged                   | D-03 |
| `--color-accent`   | `var(--color-orange-500)` → `#f97316` | unchanged                  | D-03 |
| `--color-accent-muted` | `var(--color-orange-300)`   | unchanged                   | D-03 |
| `--color-danger`   | `#e06b55`                    | unchanged                   | D-03 |
| `--color-success`  | `#5FA866`                    | unchanged                   | D-03 |
| `--color-warning`  | `#C9913C`                    | unchanged                   | D-03 |

**Color math rationale (WCAG verified 2026-05-04):**

Old pair: stone-950 (L\*=2.8) / stone-900 (L\*=9.0) — lightness delta = 6.2 L\*
units. Old border: stone-800 (L\*=15.1) — delta above panel = 6.0 L\* units.

New surface: `#232732` (L\*=15.7). Proportional panel target: L\*=21.9.

`#2c3347` lands at L\*=21.4 — closest candidate; contrast against `--color-fg`
(`#f5f5f4`) = **11.52:1** (WCAG AA requires 4.5:1 — passes with margin).

New border target: L\*=27.4. `#384158` lands at L\*=27.6 — passes.
Contrast of `#384158` against new surface `#232732` = 1.47:1 (visual
separator use only, not text — UI-only separation, no WCAG text requirement).

### 60/30/10 Contract

| Role          | Token           | New hex    | Usage (60/30/10 split)                                |
|---------------|-----------------|------------|-------------------------------------------------------|
| Dominant (60%)| `--color-surface` | `#232732` | App background, DropZone idle area, page canvas       |
| Secondary (30%)| `--color-panel` | `#2c3347` | Section cards, modal backgrounds, sticky toolbar, panel header rows |
| Accent (10%)  | `--color-accent` | `#f97316` | **Reserved for:** Optimize Assets button bg; hover/focus ring on all interactive elements; atlas-preview toggle active ring |
| Destructive   | `--color-danger` | `#e06b55` | Missing-row tints (`bg-danger/10`); left-accent bars on missing rows; MissingAttachmentsPanel and UnusedAssetsPanel header backgrounds (`bg-danger/10 border-danger/40`); warning triangle icon (`text-danger`) |

Accent is NOT used for: zebra-stripe rows, section headers, count chips,
toolbar border-style buttons (those use `border-border`), or any informational
elements. Those use `--color-border`, `--color-fg-muted`, or `--color-warning`.

Source: CONTEXT.md D-01–D-04, D-06, D-11.

### C-03: Full-row danger tint on missing rows (D-06)

**Files:**
- `src/renderer/src/panels/GlobalMaxRenderPanel.tsx` lines 428–438 (the `<tr>` element wrapping the row)
- `src/renderer/src/panels/AnimationBreakdownPanel.tsx` lines 660+ (same `<tr>` pattern)

**Change:** Add `bg-danger/10` to the `<tr>` className when `state === 'missing'`.
Use `clsx` with a literal branch. The existing name-cell `bg-danger/10 text-danger`
class stays unchanged (additive, not replaced). Left-accent bar stays unchanged.

Pattern (both panels, using existing `clsx` import):
```tsx
<tr
  className={clsx(
    'border-b border-border',
    state === 'missing' && 'bg-danger/10',
    // zebra stripe (C-07) — missing rows override zebra via specificity
  )}
>
```

**Specificity note:** `bg-danger/10` on `<tr>` takes precedence over
`even:bg-white/[0.03]` because it is a direct class, not a modifier-prefixed
class. Tailwind generates both utilities; the one appearing later in the
stylesheet wins. Since `bg-danger/10` is declared without a modifier and
`even:bg-white/[0.03]` uses the `even:` pseudo-class modifier, the base class
wins. If the executor observes any conflict in browser DevTools, use the
`clsx` ternary strategy: only apply `even:bg-white/[0.03]` when
`state !== 'missing'`.

### C-04: Warning icon sized to 16×16px (D-07)

**Files:**
- `src/renderer/src/panels/GlobalMaxRenderPanel.tsx` line 454–457

### C-07: Zebra striping (D-10)

**Files:**
- `src/renderer/src/panels/GlobalMaxRenderPanel.tsx` — `<tr>` element (around line 428)
- `src/renderer/src/panels/AnimationBreakdownPanel.tsx` — `<tr>` element (around line 660)

**Pattern:** Add `even:bg-white/[0.03]` to the `<tr>` className. Combine with
`bg-danger/10` for missing rows (C-03). Missing rows must suppress zebra via
the clsx strategy:

```tsx
<tr
  className={clsx(
    'border-b border-border',
    state === 'missing' ? 'bg-danger/10' : 'even:bg-white/[0.03]',
  )}
>
```

This ensures missing rows always show `bg-danger/10` without any chance of
even-row zebra bleeding through. Odd non-missing rows remain transparent.

**Tailwind v4 verification:** The `even:` modifier uses the `:nth-child(even)`
pseudo-class on `<tr>`. Confirm in browser DevTools that the modifier fires
correctly inside a `<tbody>`. If not, fall back to the JSX index pattern:
`rowIndex % 2 === 0 && state !== 'missing' && 'bg-white/[0.03]'` where
`rowIndex` is the map index from the existing row renderer.
