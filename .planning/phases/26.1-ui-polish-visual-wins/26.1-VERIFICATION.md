#### 1. Blue-dark color scheme appearance

**Test:** Load any Spine project. Observe the app background and panel card backgrounds.

**Test:** Load a project. Expand the window to 1440px or wider. Check GlobalMaxRenderPanel and AnimationBreakdownPanel.
**Expected:** Both panels stretch to the full window width with no visible centering gap or max-width ceiling.
**Why human:** Layout geometry requires a running renderer.

#### 3. Zebra striping and missing-row tint

**Test:** Load a project with many attachments and at least one missing PNG (or use SIMPLE_PROJECT with an atlas-less load). Scroll through the Global Max Render Source table.
**Expected:** Even rows have a subtle alternate tint. Any row for a missing attachment shows a fuller red-tinted background with no striping.
**Why human:** Visual contrast of 3% white overlay requires rendering; mutual exclusivity of tints needs UI confirmation.

#### 4. Toolbar button height uniformity

**Test:** Load a project. Look at the toolbar button cluster (Atlas Preview, Documentation, Optimize Assets, Save, Open) and the SearchBar.
**Expected:** All buttons and the search input appear exactly the same height with no visual staircase.
**Why human:** Pixel-level height equality requires a running app.

#### 5. Atlas-less chip label

**Test:** Load a project using the "Use Images Folder as Source" toggle (atlas-less mode). Observe the load-summary chip.
**Expected:** Chip shows "{n} images" (e.g. "3 images") instead of "1 atlases".
**Why human:** Requires loading an atlas-less project at runtime.

#### 6. Danger-themed panel headers

**Test:** Load a project that has missing attachment PNGs (or orphaned image files). Scroll to the MissingAttachmentsPanel and UnusedAssetsPanel headers.
**Expected:** Both headers have a red-tinted background, a triangle warning icon, and the existing left red accent bar — distinctly warning-colored compared to other panels.
**Why human:** Requires a project with missing/orphaned files to trigger the panels.

#### 7. AnimationBreakdownPanel header upgrade

**Test:** Load any project with animations. Observe the Animation Breakdown section header.
**Expected:** Header shows a small bar-chart icon, "Animation Breakdown" title at a smaller weight than before (not large h1-style), and a right-aligned chip showing "{n} animations".
**Why human:** Visual parity comparison with GlobalMaxRenderPanel header requires a running app.
