# Phase 26.2: UI polish — tab restructure + icon audit - Context

**Gathered:** 2026-05-04
**Status:** Ready for planning

<domain>
## Phase Boundary

Structural JSX change: expand the 2-tab system (`'global' | 'animation'`) to a
3-tab system (`'global' | 'unused' | 'animation'`), giving the Unused Assets
panel its own dedicated tab. Simultaneously, run UI-08 icon audit — standardize
icon style across all surfaces and add icons to tab strip buttons.

**In scope:**
- `ActiveTab` type expanded to `'global' | 'unused' | 'animation'`
- Unused tab button added to strip between Global and Animation Breakdown
- Unused tab hidden (button removed from strip) when `orphanedFiles.length === 0`
- Unused tab shows a danger-colored pill badge when `orphanedFiles.length > 0`
- `MissingAttachmentsPanel` stays above the tab strip always (unchanged position)
- `UnusedAssetsPanel` moved from "always rendered outside tabs" → conditionally
  rendered only when `activeTab === 'unused'`
- AppShell `query` state made active-tab-aware: drives whichever panel is active.
  UnusedAssetsPanel's internal `query` state lifted to AppShell; panel's own
  inline search box removed. Query persists across tab switches.
- Tab strip buttons get small inline icons (one icon per tab)
- Existing fill/solid 16px icons in MissingAttachmentsPanel and UnusedAssetsPanel
  headers converted to stroke/outline standard

**Out of scope:**
- Toolbar action button icons (Atlas Preview, Documentation, Optimize Assets,
  Save, Open) — not in this phase
- DocumentationBuilderDialog tab strip icons — not in this phase
- Draggable modals — Phase 26.3
- Any new panel logic or data changes

</domain>

<decisions>
## Implementation Decisions

### Tab restructure (LOCKED 2026-05-04)
- **D-01: Unused tab button removed entirely when `orphanedFiles.length === 0`.**
  The tab button disappears from the strip — no disabled/grayed state. Consistent
  with `UnusedAssetsPanel`'s own `return null` when empty and with ROADMAP
  "hidden when empty." Cleaner than a dead tap target.

- **D-02: Auto-redirect to 'global' when Unused tab disappears.**
  If `activeTab === 'unused'` and `orphanedFiles` drops to 0 (e.g., after
  resample), `setActiveTab('global')` fires. Mirrors existing D-50/D-74 reset
  behavior (`setActiveTab('global')` on project change). A `useEffect` watching
  `orphanedFiles.length` and `activeTab` is the clean implementation site.

- **D-03: Active-tab-aware SearchBar — query persists across tabs.**
  AppShell `query` drives the active panel's content (same pattern as Global +
  Animation currently). `UnusedAssetsPanel` loses its internal `useState`
  for `query` and gains `query: string` + `onQueryChange: (q: string) => void`
  props (matching `GlobalMaxRenderPanel` and `AnimationBreakdownPanel`). Its
  own inline search box is removed — the sticky SearchBar is the sole search
  surface. Query does NOT clear on tab switch — it persists, so the same string
  can filter whichever panel is active.

### Count badge (LOCKED 2026-05-04)
- **D-04: Pill badge with danger color tokens when N > 0.**
  When `orphanedFiles.length > 0`, a small pill chip is shown inside the Unused
  `TabButton` next to the label text. Use `bg-danger/20 text-danger` tokens (no
  raw hex) to reinforce the problem-zone theme established by Phase 26.1 for the
  UnusedAssetsPanel header. A `rounded-full text-xs font-medium px-1.5 py-0.5`
  chip is the target. The count is `orphanedFiles.length`. `TabButton` needs a
  new optional `badge?: number` prop (or pass `orphanedFiles` length inline).

### Icon style standard (LOCKED 2026-05-04)
- **D-05: Stroke/outline wins as the canonical icon style.**
  Going forward: `stroke="currentColor" strokeWidth="1.5" strokeLinecap="round"
  strokeLinejoin="round" fill="none" viewBox="0 0 20 20" className="w-5 h-5"`.
  This matches the Phase 26.1 section-header icons added to GlobalMaxRenderPanel
  and AnimationBreakdownPanel (D-12). Any NEW icon added anywhere in the app
  follows this pattern.

- **D-06: Tab strip buttons get small icons.**
  Each of the three tab buttons gets a leading icon at `w-4 h-4` (smaller than
  panel header icons to fit the compact tab strip height). Specific SVG path data
  is planner's discretion (D-06a below), but the visual intent is:
  - Global — grid/table icon (matches GlobalMaxRenderPanel section header icon)
  - Unused — warning triangle (danger-tinted, reinforces problem-zone theme)
  - Animation Breakdown — chart/bars icon (matches AnimationBreakdownPanel header)
  Icons use the same stroke style as D-05 but at `w-4 h-4`.

- **D-07: Existing fill/solid icons converted to stroke standard.**
  The following fill/16px icons are converted to stroke/outline per D-05:
  - `MissingAttachmentsPanel.tsx:65` — warning triangle `<svg viewBox="0 0 16 16" fill="currentColor">`
  - `UnusedAssetsPanel.tsx:59` — panel header icon `<svg viewBox="0 0 16 16" fill="currentColor">`
  - `GlobalMaxRenderPanel.tsx:461` — ⚠ warning icon on missing rows (Phase 25 fill icon)
  - `AnimationBreakdownPanel.tsx:703` — same ⚠ warning icon on missing rows (Phase 25 fill icon)
  Size: these contextual icons may stay at `w-4 h-4` if the surrounding element
  constrains height (planner to determine per site — do not change icon size if
  it would create layout shifts).

### Claude's Discretion
- **Specific SVG path data for tab icons (D-06a):** Research will select from the
  existing inline SVG inventory in the codebase or author compatible paths.
  Constraint: stroke/outline style only; `viewBox="0 0 20 20"` or `"0 0 16 16"`
  as fits `w-4 h-4` render target; must look visually coherent at small size.
- **Converted icon size per-site (D-07a):** After conversion from fill to stroke,
  retain the same `w-4 h-4` class if the surrounding element constrains the space
  (e.g., inside a table row or a panel header with fixed height). Only upsize to
  `w-5 h-5` if the context clearly has room. Prioritize no layout shift.
- **`TabButton` badge prop shape:** Either a dedicated `badge?: number` prop or
  the badge element inlined in the call site using `orphanedFiles.length`. Either
  works; consistency with the existing `isActive: boolean` prop pattern is the
  guide.

</decisions>

<code_context>
## Existing Code Insights

### Reusable Assets
- `TabButton` function at `AppShell.tsx:1741` — existing 2-weight contract with
  `isActive: boolean`, `onClick`, `role="tab"` ARIA. Needs a new optional `badge`
  slot and a leading icon slot added.
- `clsx` — already imported in AppShell and all panel files; use for conditional
  badge visibility.
- `DimsBadge.tsx` — `w-4 h-4` SVG at stroke style is the icon size reference
  for tab buttons. Pill badge token pattern: `bg-danger/20 text-danger`.

### Established Patterns
- **Tailwind v4 literal-class discipline**: no template string interpolation in
  className. All class strings must be literals or clsx conditional branches with
  literal values. The badge `bg-danger/20 text-danger rounded-full` must appear
  as verbatim strings.
- **Token-only colors**: `bg-danger/20`, `text-danger` — never raw hex in JSX.
- **`useEffect` for activeTab guard**: D-02 redirect uses a `useEffect` watching
  `[orphanedFiles.length, activeTab]`. This mirrors the existing pattern for
  `setActiveTab('global')` on project change.
- **Conditional panel rendering**: `{activeTab === 'global' && <GlobalMaxRenderPanel>}`
  pattern at AppShell:1538 and :1562. UnusedAssetsPanel moves into this pattern.

### Integration Points
- `AppShell.tsx` is the single change hub for the tab type, tab state, tab strip
  render, SearchBar query threading, and panel conditional rendering.
- `UnusedAssetsPanel.tsx` needs prop interface additions + removal of internal
  query state + removal of inline search box. The `return null` when empty at
  line 47 may be retained as a safety guard or removed since the tab is now
  hidden when empty — planner to confirm.

</code_context>

<specifics>
## Specific Ideas

- **Pill badge visual target**: `bg-danger/20 text-danger rounded-full text-xs
  font-medium px-1.5 py-0.5 ml-1.5` — small danger chip next to the "Unused"
  text in the tab button. Danger color reinforces that orphaned files are a
  problem zone (consistent with the Phase 26.1 danger-themed UnusedAssetsPanel
  header bar).
- **Tab strip after restructure**: `[ Global ] [ Unused ●12 ] [ Animation Breakdown ]`
  where `●12` is the pill badge. When orphaned files are 0: `[ Global ] [ Animation Breakdown ]`.
- **SearchBar persistence rationale**: query persists across tab switches so
  that an animator searching "btn_" on Global can switch to Unused and
  immediately see whether any orphaned files contain "btn_" — useful cross-check
  in large rigs.

</specifics>
