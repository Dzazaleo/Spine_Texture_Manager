## Implementation Decisions

### Tab strip relocation (LOCKED 2026-05-04)
- **D-01: Tab strip moves from inside main toolbar to a dedicated sub-toolbar row.**
  Sketch-001 variant A is the locked design. The existing 2-tab strip currently rendered inside the main toolbar `<header>` ([AppShell.tsx ~line 1327-1339](../../../src/renderer/src/components/AppShell.tsx#L1327-L1339)) is extracted into a sibling `<nav role="tablist">` row immediately below the toolbar, separated by `border-t border-border`. Active tab keeps the existing 2px orange underline indicator (`-bottom-px h-[2px] bg-accent`). See `.continue-here.md` AP-01 for the structural diagram and the prior-revert history.

- **D-02: ActiveTab type stays `'global' | 'animation'`.** No 3-tab expansion. No `'unused'` literal added. AppShell:78 stays as-is. AppShell:162 default value `'global'` stays as-is.

### Tab icons (LOCKED 2026-05-04)
- **D-03: Verbatim panel-header SVG reuse, sized down.**
  Tab icons reuse the exact `<rect>` / `<path>` data from the panel section-header icons. No restyling, no path-data divergence.
  - Global tab icon: copy children of [GlobalMaxRenderPanel.tsx:836-839](../../../src/renderer/src/panels/GlobalMaxRenderPanel.tsx#L836-L839) (filmstrip — `<rect x="2" y="6" width="16" height="8" rx="1" />` + `<path d="M5 6 v3 M8 6 v2 M11 6 v3 M14 6 v2 M17 6 v3" />`).
  - Animation Breakdown tab icon: copy [AnimationBreakdownPanel.tsx:415](../../../src/renderer/src/panels/AnimationBreakdownPanel.tsx#L415) (bar-chart — `<path d="M3 17 V10 M8 17 V6 M13 17 V12 M18 17 V4" />`).
  - Wrapping `<svg>` attributes match D-04 stroke standard; only the `className` differs (`w-4 h-4` instead of `w-5 h-5`).
  - Icons sit left of the label text, separated by `gap-1` (4px).
  - Color: `currentColor` — inherits from the tab button text color (active orange / inactive muted).

### Stroke icon standard (LOCKED 2026-05-04)
- **D-04: Canonical stroke icon attributes.** Every icon authored or modified in this phase follows:
  ```
  stroke="currentColor"
  strokeWidth="1.5"
  strokeLinecap="round"
  strokeLinejoin="round"
  fill="none"
  viewBox="0 0 20 20"
  className="w-5 h-5"   (or "w-4 h-4" per site constraint)
  aria-hidden="true"
  ```
  Carries forward Phase 26.1 D-12 stroke standard. Tab icons + the 4 conversion sites all conform.

### Fill → stroke conversion (LOCKED 2026-05-04)
- **D-05: Convert 4 fill icons to stroke.** Sites: MissingAttachmentsPanel:65, UnusedAssetsPanel:59, GlobalMaxRenderPanel:461, AnimationBreakdownPanel:703. All 4 retain `w-4 h-4` (no upsize — these sit in space-constrained alert-bar headers and table cells; upsizing would create layout shifts).

- **D-06: Single canonical stroke-triangle path used by ALL warning-triangle instances.**
  The 4 conversion sites in D-05 ALL render the same warning-triangle SVG. They MUST share one identical path (no per-site divergence). Canonical path:
  ```svg
  <svg viewBox="0 0 20 20" stroke="currentColor" strokeWidth="1.5" strokeLinecap="round" strokeLinejoin="round" fill="none" className="w-4 h-4" aria-hidden="true">
    <path d="M10 3 L17 16 H3 Z" />
    <path d="M10 8 V12" />
    <circle cx="10" cy="14.5" r="0.5" />
  </svg>
  ```
  - Outer triangle: `M10 3 L17 16 H3 Z` — equilateral-ish, top vertex at (10,3), bottom at (3,16) and (17,16).
  - Inner exclamation stem: `M10 8 V12` — vertical stroke.
  - Inner exclamation dot: small filled circle at (10, 14.5) radius 0.5 — rendered as a stroked circle (the 1.5 stroke width fills the small circle visually). If the small-radius circle looks off, planner may substitute `<path d="M10 14.5 V14.5" />` (a zero-length stroke renders as a dot via `strokeLinecap="round"`).
  - Planner discretion (D-06a): the exclamation dot's exact rendering — `<circle r="0.5">` or zero-length stroke path — is whichever produces a visually consistent dot at 16×16 pixels in dark-mode. Both options preserve the single-path-shared invariant if applied uniformly.
  - All 4 conversion sites must paste this exact `<svg>` block (or import a shared `WarningTriangleIcon` component, planner's call — see D-07).

- **D-07: Whether to extract `WarningTriangleIcon` as a shared component is planner's call.**
  Either inline the same `<svg>` block at all 4 sites (simpler, no new file) or extract to `src/renderer/src/components/icons/WarningTriangleIcon.tsx` (DRY, single source of truth at code level). The single-source-of-truth INVARIANT is fixed; the implementation form is open. If extracted, the component must accept `className?: string` prop so callers can pass `text-danger` / `text-fg` / `w-4 h-4` etc.

### Wording rename (LOCKED 2026-05-04)
- **D-08: Rename "orphaned" → "unused" on the 2 user-visible strings only.**
  - [UnusedAssetsPanel.tsx:71](../../../src/renderer/src/panels/UnusedAssetsPanel.tsx#L71): `{count} orphaned file{plural}` → `{count} unused file{plural}`
  - [UnusedAssetsPanel.tsx:52](../../../src/renderer/src/panels/UnusedAssetsPanel.tsx#L52): `aria-label="Orphaned image files"` → `aria-label="Unused image files"`
  - All other "orphan" strings in the codebase (internal types `OrphanedFile`, prop names `orphanedFiles`, variable names `filteredOrphans`, comments, AppShell.tsx prop pass-through) are **NOT** renamed. Internal identifier churn is out of scope.

### Inline search removal (LOCKED 2026-05-04)
- **D-09: Delete the inline `<SearchBar>` inside UnusedAssetsPanel.**
  Remove [UnusedAssetsPanel.tsx:87-93](../../../src/renderer/src/panels/UnusedAssetsPanel.tsx#L87-L93) (the wrapping `<div>` + the `<SearchBar>` import usage). Remove the supporting state: `const [query, setQuery] = useState('')` (line ~31) and the `useMemo` block computing `filteredOrphans` (lines 39-43). Replace `filteredOrphans` references in the table render with `orphanedFiles` directly (no filtering). The `(no matches)` empty-state cell is no longer reachable — remove the conditional and unconditionally render `orphanedFiles.map(...)`.
  - The unused-files list is bounded (atlas-less mode at most yields a small handful of orphans typical for a Spine project); a filter input is not needed at this scale.
  - The toolbar's main SearchBar does NOT drive this list — the unused-files alert bar is independent of the active tab.

### Claude's Discretion
- **D-06a:** Exclamation-dot rendering inside the warning triangle (`<circle r="0.5">` vs zero-length stroke path) — planner picks whichever renders cleanly at 16px in dark-mode.
- **D-07:** Inline `<svg>` at 4 sites vs extract `WarningTriangleIcon` component — planner's call. Single-source-of-truth invariant is fixed.
