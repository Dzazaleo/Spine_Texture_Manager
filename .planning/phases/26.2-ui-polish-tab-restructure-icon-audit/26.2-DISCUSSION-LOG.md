## Unused tab when empty

### Question 1: What happens when orphanedFiles.length === 0?

| Option | Description | Selected |
|--------|-------------|----------|
| Remove entirely | Tab button disappears from strip. Matches ROADMAP "hidden when empty." Consistent with UnusedAssetsPanel's own `return null`. | ✓ |
| Keep, show disabled | Tab button stays but grayed out / pointer-events-none. Dead tap trap; inconsistent with panel self-hide. | |

**User's choice:** Remove entirely (recommended)
**Notes:** Unanimous preference for clean removal — no dead tabs.

### Question 2: Auto-redirect when Unused tab disappears?

| Option | Description | Selected |
|--------|-------------|----------|
| Fall back to Global tab | `setActiveTab('global')` when orphanedFiles drops to 0. Consistent with D-50/D-74 reset. | ✓ |
| Stay, show empty state | Keep activeTab = 'unused' even when button is hidden; render empty state. | |

**User's choice:** Fall back to Global tab (recommended)

## SearchBar with 3 tabs

### Question 1: What does the SearchBar do on the Unused tab?

| Option | Description | Selected |
|--------|-------------|----------|
| Active-tab-aware | AppShell query drives whatever panel is active. Unused gets query prop. Remove Unused's own inline search box. | ✓ |
| Keep split | AppShell query drives Global+Animation only. Unused keeps own inline search box inside tab content. | |

**User's choice:** Active-tab-aware (recommended)

### Question 2: Does query clear on tab switch?

| Option | Description | Selected |
|--------|-------------|----------|
| Clear on tab switch | `setQuery('')` on tab change. Prevents "CIRCLE" from filtering orphaned files accidentally. | |
| Persist across tabs | Same query string applies to active panel. Useful cross-check across tabs. | ✓ |

**User's choice:** Persist across tabs
**Notes:** Persistence allows the animator to search "btn_" on Global then switch to Unused to see if any orphaned files also match — useful in large rigs.
