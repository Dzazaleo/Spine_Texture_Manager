> blocking anti-patterns must be acknowledged in your own words before proceeding.

## Critical Anti-Patterns

| ID | Severity | Anti-Pattern | History |
|----|----------|--------------|---------|
| AP-01 | **blocking** | Placing the Global / Animation Breakdown tab strip inside the main toolbar action row (alongside file badge, Atlas Preview, Documentation, Optimize Assets, Save, Open buttons). | Two prior reverts: revert commit `c13ca9c` (2026-05-04) + earlier revert. Both attempts inlined tabs into the toolbar; both got reverted. The user's locked design (sketch-001 variant A) places tabs in a **dedicated sub-toolbar row immediately below the main toolbar**, separated by a `border-t border-border`. |

## AP-01 — How to comply

**Required structure (sketch-001 variant A):**

```
┌─ Main toolbar (existing — unchanged) ────────────────────────────┐
│ [.stmproj] [counts] [SearchBar] [☑ Source] [Atlas] [Doc] [...]   │
└──────────────────────────────────────────────────────────────────┘
┌─ Sub-toolbar (NEW — tabs only) ──────────────────────────────────┐
│ [⊞ Global]  [▦ Animation Breakdown]                              │
└──────────────────────────────────────────────────────────────────┘
[ alert bars: missing PNG, unused files (existing — unchanged) ]
[ active panel content ]
```

- Sub-toolbar is its own `<nav role="tablist">` row, NOT a child of the main toolbar `<header>`.
- Active tab indicator stays as the existing 2px orange underline (`-bottom-px h-[2px] bg-accent`).
- See: [.planning/sketches/001-tab-bar-placement/index.html](../../sketches/001-tab-bar-placement/index.html) variant A — the reference rendering.
- See: [.planning/sketches/001-tab-bar-placement/README.md](../../sketches/001-tab-bar-placement/README.md) — design question + verdict.

**Required acknowledgement template** (write inline before planning/executing):

> AP-01 is the toolbar-inlined tab anti-pattern. It manifested as two
> reverted attempts (c13ca9c + earlier) that placed the tab buttons inside
> the main toolbar `<header>` row. The structural mechanism that prevents
> recurrence is sketch-001 variant A's locked layout: a dedicated
> `<nav role="tablist">` element rendered as a sibling row below the main
> toolbar `<header>`, separated by `border-t border-border`, never as a
> descendant of the toolbar.
