- **D-10 (supersedes D-09, locked 2026-04-28):** Use Option C from `12.1-01-BLOCKED.md` (originally framed as fallback under D-09, now promoted to primary path) — author a small post-build synthesizer script `scripts/emit-latest-yml.mjs` that runs AFTER `electron-builder` exits 0 in each `build:mac` / `build:win` / `build:linux` npm script, reads the produced installer artifact (`.dmg` / `.exe` / `.AppImage`) byte content, computes SHA-512 + byte size with `node:crypto` + `node:fs.statSync`, and writes `release/latest-{mac,win,linux}.yml` to the schema electron-updater@6.x consumes. `electron-builder.yml` `publish:` reverts permanently to `null` (NOT `never`) — `null` cleanly suppresses publisher-chain construction, which is the load-bearing safety property (no race possible). The static `build/app-update.yml` + `extraResources` mapping from commit `679cffc` stays in place unchanged. Two roles that `electron-builder.yml` was being asked to play simultaneously (suppress publisher race AND emit feed metadata) are now cleanly separated: `publish: null` handles the first; the synthesizer script handles the second.

  **Schema for `release/latest-{mac,win,linux}.yml` (per electron-updater@6.x docs + the existing repo's pre-Phase-12 `release/latest-mac.yml` carried over from Phase 11; both already in canonical_refs):**

  ```yaml
  version: 1.1.0-rc1                                # MUST match package.json `version` exactly
  files:
    - url: <installer filename>                     # e.g., "Spine Texture Manager-1.1.0-rc1-arm64.dmg"
      sha512: <128-char base64 SHA-512>             # computed from the installer byte content
      size: <positive integer>                      # installer size in bytes (matches `wc -c`)
  path: <same as files[0].url>                      # legacy top-level mirror for older electron-updater
  sha512: <same as files[0].sha512>                 # legacy top-level mirror
  releaseDate: '<ISO 8601 string>'                  # `new Date().toISOString()`
  ```

  Both the modern `files[]` array and the legacy top-level `path`/`sha512` mirror are emitted (electron-updater@6.x reads `files[]` first but falls back to top-level for backward compatibility; emitting both is the safest contract).

  **Synthesizer script spec (`scripts/emit-latest-yml.mjs`):**

  - Location: `scripts/emit-latest-yml.mjs` (matches existing `scripts/cli.ts` + `scripts/probe-*.ts` convention; `.mjs` ESM for native `node:crypto`/`node:fs`/`node:path` imports)
  - Invocation: `node scripts/emit-latest-yml.mjs --platform=mac|win|linux` chained into the 3 npm scripts (`build:mac` becomes `electron-vite build && electron-builder --mac dmg --publish never && node scripts/emit-latest-yml.mjs --platform=mac` — same pattern for win/linux)
  - Input: scans `release/` for the platform-appropriate installer (`*.dmg` for mac, `*.exe` for win, `*.AppImage` for linux); fails fast if the file is missing or if multiple matches exist (force the build pipeline to commit to one installer per platform)
  - Output: writes `release/latest-{mac,win,linux}.yml` via the atomic Pattern-B write (`.tmp` + `fs.rename`) to match the project's established invariant on file-system writes (CONTEXT.md `<code_context>` Established Patterns)
  - Idempotency: re-running on the same `release/` produces byte-identical YAML (modulo `releaseDate` — which is freshly stamped per run; this matches electron-builder's own behavior)
  - Error handling: explicit error messages on missing installer, multiple matches, or SHA-512/size compute failures; non-zero exit propagates to fail the npm script
  - Tests: vitest integration test under `tests/integration/emit-latest-yml.spec.ts` with a small fixture installer (any binary file ≥ 1KB) asserting (a) emitted YAML parses, (b) all 6 fields present + well-formed, (c) `files[0].sha512` matches the fixture's `crypto.createHash('sha512').update(buffer).digest('base64')` reference, (d) `files[0].size` matches `fs.statSync(fixture).size`. Test uses a temp directory so it can run on any platform.

If both pass → atomic-commit the synthesizer + package.json + electron-builder.yml `publish:` revert (a single fix commit) + author SUMMARY. If EITHER check fails → halt; do NOT commit; revert working-tree changes; re-enter `/gsd-discuss-phase 12.1` to escalate to **Option D** (`provider: generic` + hosted feed URL). Skip Option B entirely (excluded). Record any new empirical evidence in a follow-up `12.1-01-BLOCKED-D-10.md` document modeled on the existing two BLOCKED docs.
