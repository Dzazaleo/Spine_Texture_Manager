## Resolutions (delegated → Claude's recommended defaults)

### Publish-race fix architecture (Gap 1 → D-01)

| Option | Pros | Cons | Selected |
|--------|------|------|----------|
| (a) `publish: null` + static `build/app-update.yml` via `extraResources` | One source of truth; restores Phase 11 era working state; HIGH research confidence | Requires hand-authored static YAML file | ✓ |
| (b) `--config.publish=null` CLI override on build:* scripts | More surgical (file untouched) | Split source-of-truth (YAML says github, CLI overrides null) — confusing for future maintainers | |
| (c) Keep `publish: github` + drop `GH_TOKEN` from build env | Minimal YAML change | Rebuilds failure mode (i): publisher constructor throws on missing token; risk of build abort | |

**Rationale:** Plan 12.1-01 already implements Option (a). Research §Summary explicitly recommends (a). Restores the architecture that was working through Phase 11 Plan 11-02 close.

### Tag-push safety + rc2/rc3 sequencing (Gap 3+8 → D-02 + D-03)

| Option | Pros | Cons | Selected |
|--------|------|------|----------|
| Pre-flight workflow_dispatch dry-run BEFORE tagging rc2 | Catches publish-race regression in non-polluting context (no draft Release, no tag pollution); ~10 min CI wait is cheap insurance | Slight delay before rc2 push | ✓ (D-02) |
| Tag rc2 directly without pre-flight | Faster path to live verification | 3 prior tag-push attempts already polluted the release surface; risk of 4th polluted tag | |
| rc2 → install on ALL 3 OSes → THEN rc3 (atomic) | Single observation window for rc2→rc3 detection across all hosts; per RESEARCH Pitfall 3 | Requires coordination across OSes | ✓ (D-03) |
| rc2 → install on macOS first → fix issues → Linux → Windows → tag rc3 (incremental per-OS) | Lower coordination cost | Could miss rc2→rc3 detection on later OSes if rc3 tagged before rc2 installed there | |

**Rationale:** D-02 — User has 3 documented polluted-tag incidents (run IDs 25017095851 / 25017351602 / 25017624868); risk-aversion warranted. D-03 — RESEARCH §Pitfall 3 explicitly warns against per-OS incremental sequencing. Plan 12.1-02 Task 2 enforces atomic gating.
