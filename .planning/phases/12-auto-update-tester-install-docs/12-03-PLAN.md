<objective>
Fix F1 (the Phase 11 spillover Windows runtime bug) at its single audit site: `src/renderer/src/modals/AtlasPreviewModal.tsx:116`. The current code naively concatenates `app-image://localhost` with `encodeURI(absolutePath)`, which on Windows glues the drive-letter `C:` onto `localhost` to produce host `localhostc` and a 404'd image fetch. RESEARCH §F1 audit verified this is the **only** site constructing `app-image://` URLs from filesystem paths — single-site fix.

The fix is a 2-line change at line 116, swapping string concat for `pathToFileURL().pathname` from `node:url`. Because `node:url` is a Node-only module and the renderer runs sandboxed, `pathToFileURL` is exposed via a new preload bridge `window.api.pathToImageUrl(absolutePath: string): string`. A Windows-runtime regression test runs in the 12-02 expanded matrix to catch any future drift.

Purpose: Without F1's fix, Atlas Preview (a primary user-facing feature) is silently broken on Windows. The Phase 12 success criterion (testers using the Windows installer can verify Atlas Preview works as a basic sanity check) cannot be claimed without this fix. Lands in Wave 2 in parallel with 12-01/12-04/12-05 because it touches independent files and is small (~4 file edits, ~30 lines net).

Output: A working Atlas Preview on Windows, with regression coverage running on every CI run via the 12-02 matrix expansion.
</objective>
