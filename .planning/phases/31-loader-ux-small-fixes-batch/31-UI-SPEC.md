### Accent reservation (10% rule)

**Accent is reserved for, in this phase only:**
1. Hover border + hover text on bulk-action buttons (sub-feature B) — matches every other `h-8` toolbar button.
2. Focus-visible outline ring on bulk-action buttons (sub-feature B) — `outline-2 outline-accent outline-offset-2`.

**Accent is explicitly NOT used for:**
- The disabled source-toggle menu item (sub-feature A) — disabled state suppresses accent. The hover affordance from the parent menu container at `AppShell.tsx:1737` (`hover:border-accent hover:text-accent`) is preserved on the OUTER chip but the disabled INNER menu item must not inherit it (use `disabled:` Tailwind variants or block hover with pointer-events).
- The DropZone advisory text (sub-feature C) — `fg-muted` only. The existing drag-over ring (`ring-2 ring-accent`) is suppressed when `isElevated && process.platform === 'win32'` because handlers are no-op'd.
- The ExtrapolationIcon (sub-feature D) — currently `text-white` at `GlobalMaxRenderPanel.tsx:574` and `AnimationBreakdownPanel.tsx:800`; this is preserved verbatim. The icon's color is locked; only its tooltip-rendering mechanism changes.

### Color contract for disabled-state communication (sub-feature A)

The disabled "Use Atlas as Source" / "Use Images Folder as Source" menu item is communicated through **three redundant signals** (matches the v1.0 D-91 redundant-encoding pattern from override visualization):

1. **Text color:** `text-fg-muted` (50% reduced contrast from `text-fg`).
2. **Cursor:** `cursor-not-allowed` (mouse pointer changes; no click animation).
3. **Pointer events:** `pointer-events-none` on the click target — OR — `disabled` attribute if the menu-item is migrated from `<button>`-shaped span to a true `<button disabled>` (planner discretion; both are acceptable).

The existing `hover:bg-surface` from `AppShell.tsx:1763` MUST be suppressed on the disabled menu item. Use Tailwind `disabled:` variants if migrating to `<button disabled>`; use a className-conditional otherwise.

## Interaction Contract

### Sub-feature A — Disabled menu item

| State | Visual | Interactive |
|-------|--------|-------------|
| **Atlas mode + `hasAtlasFile === true`** (currently in atlas-source mode, atlas file exists; menu entry would say "Use Images Folder as Source" if `hasImagesDir === false` it disables — otherwise enabled) | Default `text-xs text-fg hover:bg-surface` | Click swaps loaderMode |

| **Atlas mode + alt source absent** (`atlasPath !== null` AND `hasImagesDir === false`) | `text-xs text-fg-muted cursor-not-allowed`; native `title="No images/ folder found in this project's folder"`; no hover bg change | `disabled` (button) or `pointer-events-none` (span) — clicks are no-ops |
| **Atlas-less mode + alt source absent** (`atlasPath === null` AND `hasAtlasFile === false`) | `text-xs text-fg-muted cursor-not-allowed`; native `title="No .atlas file found in this project's folder"`; no hover bg change | `disabled` (button) or `pointer-events-none` (span) — clicks are no-ops |

**ARIA contract:**
- The element MUST set `aria-disabled="true"` (works for both `<button disabled>` and `<span pointer-events-none>` shapes).
- Native `title=` attribute provides the visible-on-hover tooltip AND supplies the disabled reason to screen readers (most modern AT read `title` after `aria-label`).
- Outer chip's hover affordance (`hover:border-accent hover:text-accent` at `AppShell.tsx:1737`) is **preserved** — the chip still "lights up" on hover so the user knows the menu is interactive; only the inner item is disabled.
- `tabindex` — if the menu item is migrated to `<button disabled>`, browsers naturally skip it in tab-order (correct behavior). If it stays a `<span>` with `pointer-events-none`, ensure NO `tabindex="0"` is set; default `<span>` is non-tabbable.
