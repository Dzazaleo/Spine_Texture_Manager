### Task 3: Wire DimsBadge into both panels + AppShell

- `GlobalMaxRenderPanel.tsx`: `loaderMode` prop added; native badge JSX replaced with `<DimsBadge>` using pre-cap effectiveScale
- `AnimationBreakdownPanel.tsx`: byte-symmetric mirror; loaderMode threaded through AnimationCardProps → AnimationCard → BreakdownTableProps → BreakdownTable → BreakdownRowItemProps → BreakdownRowItem
- `AppShell.tsx`: `loaderMode={loaderMode}` added to both panel JSX usages

**Pre-cap scale deviation:** Task 3 revealed that `EnrichedRow.effectiveScale` is already capped by `sourceRatio` inside `computeExportDims`. Passing this post-cap value to `deriveIsCapped` would make `deriveIsCapped(row, 0.67)` return `0.67 > 0.67 = false` even when no override is applied — suppressing the cap suffix incorrectly. Fix: compute pre-cap scale inline as `Math.min(safeScale(row.override !== undefined ? row.override / 100 : row.peakScale), 1.0)` before passing to DimsBadge. This is a Rule 1 auto-fix.

Commit: `9a81028`

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Pre-cap effectiveScale causes cap-binding false negative**
- **Found during:** Task 3 wiring (confirmed during Task 4 test debugging)
- **Issue:** `EnrichedRow.effectiveScale` is post-sourceRatio-cap (applied inside `computeExportDims`). Passing this to `deriveIsCapped` made the cap check `0.67 > 0.67 = false` even when the actual desired scale (1.0) would have triggered the cap.
- **Fix:** Compute pre-cap scale inline in Row/BreakdownRowItem render: `Math.min(safeScale(row.override !== undefined ? row.override / 100 : row.peakScale), 1.0)`, pass to DimsBadge as `effectiveScale`.
- **Files modified:** GlobalMaxRenderPanel.tsx, AnimationBreakdownPanel.tsx
- **Commit:** 112bfd0 (included in Task 4 commit)

**2. [Rule 1 - Bug] Auto-expanded setup-pose card collapsed by erroneous click**
- **Found during:** Task 4 sibling-symmetry test debugging
- **Issue:** AnimationBreakdownPanel auto-expands the setup-pose card via `userExpanded = new Set(['setup-pose'])`. The test was clicking the card header to "expand" it, which actually collapsed it.
- **Fix:** Removed the `fireEvent.click(cardHeader)` — card is already expanded on mount.
- **Files modified:** tests/renderer/anim-breakdown-virtualization.spec.tsx
- **Commit:** 112bfd0

**3. [Rule 3 - Blocking] Atlas format mismatch in Task 5 script design**
- **Found during:** Task 5 implementation
- **Issue:** Plan assumed atlas had per-region `size: W, H` lines. Actual `SIMPLE_TEST.atlas` uses `bounds:x,y,w,h` format — no `size:` per region. Script would have exited with ERROR.
- **Fix:** Script mutates `bounds` w,h components of the SQUARE region line instead of a `size:` line. Preserves x,y coords. Confirmed working.
- **Files modified:** scripts/fixture-atlas-source-drift.mjs
- **Commit:** 7fed0d3
