## Phase 22 Cap Math — Vestigiality Confirmation

After G-01 D-01's unified model, `actualSourceW === region.originalWidth` in atlas-source mode. In atlas-less mode, `actualSourceW` comes from PNG IHDR. In BOTH modes, `sourceW === actualSourceW` after G-01 (sourceW is derived from the same atlas.region.originalWidth that now populates actualSourceW).

Therefore, the Phase 22 cap math at `src/core/export.ts:200-217`:
```typescript
const sourceRatio = Math.min(actualSourceW / sourceW, actualSourceH / sourceH);
const cappedEffScale = Math.min(downscaleClampedScale, sourceRatio);
```

When `actualSourceW === sourceW`, `sourceRatio = 1.0` always. `cappedEffScale = downscaleClampedScale` (cap is a no-op). The `isCapped` flag (line 214: `downscaleClampedScale > sourceRatio`) fires only when `downscaleClampedScale > 1.0`, which is impossible (clamp at ≤1.0 happens before this). So `isCapped` is always `false` after G-01.

**Flag for plan 22.1-03:** The cap math and `peakAlreadyAtOrBelowSource` branch at `export.ts:200-217` are vestigial after G-01. Plan 22.1-03 (export partition restructure) should DELETE this code per CONTEXT D-06 ("Phase 22 D-04 REVISED predicate SUPERSEDED by Phase 22.1 D-06"). The simpler `isPassthrough = (outW === sourceW AND outH === sourceH)` predicate is the correct partition.
