## G-01 Badge mode strategy (chat-mode discussion)

| Option | Description | Selected |
|--------|-------------|----------|
| Gate badge to atlas-less mode only — minimal fix | Badge predicate adds `&& loaderMode === 'atlas-less'`. Loader/export underlying behavior UNCHANGED. Doesn't fix root cause. | |
| Gate badge AND fix loader/export to ignore images/ in atlas-source mode (B-strict) | Atlas-source loader skips per-region PNG header reads; export pulls from atlas page extracts via existing fallback. Restores mental model fully. | ✓ |
| Gate badge + add explicit "use images/ as override in atlas mode" toggle | Most flexible but most surface; likely overkill. | |

**User's choice:** B-strict (gate + fix loader/export to ignore images/ in atlas-source mode)
**Notes:** User clarified that atlas-source mode and atlas-less mode should be mutually exclusive (no dual-source ambiguity). User also confirmed projects will NEVER use rotated atlas regions — non-negotiable for this app's contract — so the B-strict regression risk (rotated regions hitting `rotated-region-unsupported` error in atlas-extract path) is acceptable AND should be moved to load-time rejection (G-01b NEW). Mid-discussion the user surfaced an additional question: what about TexturePacker scale<1 atlas-source case? Resolved by extending B-strict with the unified actualSource model — atlas-source mode populates `actualSourceW/H` from `atlas.region.originalWidth/Height` (NOT from images/), so drift is detected when atlas-orig < JSON canonical (TexturePacker scale<1 scenario) without reading images/ at all.

## G-07 Override-on-drifted-row dim label semantics (chat-mode discussion)

| Option | Description | Selected |
|--------|-------------|----------|
| actualSource → target, always for drifted rows | Drifted rows render `actualSourceW × actualSourceH → outW × outH`. Consistent: 'source' = what's on disk. | (functionally yes — see notes) |
| canonical → target | Drifted rows render `canonicalW × canonicalH → outW × outH`. Surfaces canonical as user-facing 'source'. | |
| Both shown — `actualSource (canonical) → target` | Maximally informative but visually busier. | |

**User's choice:** Implicit — by changing the override semantic to "% of source-on-disk" (D-05) and unifying sourceW === actualSourceW everywhere (D-01), the dim label naturally renders `sourceW → outW` which equals `actualSource → target` in atlas-less mode. Option (a) achieved structurally rather than via explicit label routing.
**Notes:** User asked "when source is smaller than canonical, the canonical dims should be purely informative. So 50% of a source 670x670 that used to be 1000x1000 should not be 500x500 but 375x375 [sic — math is 335]. Is this confusing? to me seems simpler. How does this impact the math we already have established? Is it easy to implement?" Investigation revealed the existing math in atlas-less mode ALREADY interprets override as % of sourceW = % of actualSource (Phase 22 D-02 wording was misleading; code did the right thing). No math change needed — only semantic label correction (D-05). User then confirmed atlas-source mode (after G-01 B-strict + unified model) follows the same pattern: override = % of atlas-region-orig dim, which equals % of actualSource in the unified model.
