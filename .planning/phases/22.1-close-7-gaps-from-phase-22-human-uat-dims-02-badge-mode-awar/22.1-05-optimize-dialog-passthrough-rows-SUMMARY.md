## What Changed

### `src/renderer/src/modals/OptimizeDialog.tsx`

**PreFlightBody — passthroughCopies block (final lines ~497-526):**

Replaced the previous single-expression `.map((row) => (...))` with a block body `.map((row) => { ... return (...) })` that:
1. Computes `sourceDimW = row.actualSourceW ?? row.sourceW` and `sourceDimH` (CHECKER FIX preserved).

3. Dropped the `(already optimized)` text from the dim-label span (G-05).
4. Added `{row.isCapped && <span className="ml-2 text-fg-muted text-[10px]">(capped)</span>}` after the COPY chip (G-07 D-07).
5. Preserved `opacity-60` on the `<li>` container and `text-fg-muted` on the path/dim spans (Phase 6 D-109).

**InProgressBody — dimText construction (lines ~585-587):**

Updated `dimText` to use `${sourceLabel} → ${row.outW}×${row.outH}` for ALL rows (passthrough and resize). Previously passthrough rows used `${sourceLabel} (already optimized)`. Now G-05 + G-06 D-08 parity extends to the in-progress view. The COPY chip in the in-progress row continues to communicate byte-copy semantic.
