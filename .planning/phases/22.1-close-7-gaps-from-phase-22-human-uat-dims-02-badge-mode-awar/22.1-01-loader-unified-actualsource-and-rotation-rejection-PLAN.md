<objective>
Implement the unified actualSource model (G-01 D-01), load-time rotation rejection (G-01b D-03), AND wire the existing MissingImagesDirError typed error into the no-images-folder branch (G-08 D-09) — all in src/core/loader.ts. In atlas-source mode the loader stops reading images/ PNG headers and instead derives actualSourceW/H from atlas.region.originalWidth/originalHeight (which spine-core 4.2 auto-backfills from packed width/height when no orig: line is present). The loader also scans for rotated regions during load and throws a typed RotatedRegionUnsupportedError before skeleton parsing completes. Finally, when the user has explicitly toggled atlas-less mode and the images/ folder is missing, the loader throws MissingImagesDirError (precise actionable message) instead of the misleading shared AtlasNotFoundError. Atlas-less mode behavior for the actualDimsByRegion path is preserved verbatim.

Purpose: This is the foundational loader-level simplification that the rest of Phase 22.1 depends on. After this plan, sourceW === actualSourceW in BOTH modes (atlas-source: from atlas region orig; atlas-less: from PNG header), which makes the Phase 22 cap math vestigial and allows the partition predicate simplification in plan 22.1-03. It also fixes the user's original UAT bug (full-scale atlas + manually-shrunk images/ producing spurious badges) by ignoring images/ in atlas-source mode. The G-08 fold-in piggy-backs on the same loader file + the same `./errors.js` import line touched by the rotation work, eliminating the parallel-execution conflict that would arise from a separate plan.
