### CR-001: `computeExportDims` override missing source-ratio adjustment

**Files modified:** `src/renderer/src/lib/export-view.ts`, `src/renderer/src/panels/GlobalMaxRenderPanel.tsx`, `src/renderer/src/panels/AnimationBreakdownPanel.tsx`
**Commit:** 029f6e9
**Applied fix:** Added `canonicalW?: number` and `canonicalH?: number` optional parameters to `computeExportDims`. The override `rawEffScale` formula now multiplies by `(sourceW / (canonicalW ?? sourceW))` to match the `buildExportPlan` formula exactly, converting the override from source-relative to canonical-relative so `ceil(canonicalW × effScale)` yields the correct source-relative output. Both panel callers (`enrichWithEffective` in GlobalMaxRenderPanel and AnimationBreakdownPanel) updated to pass `row.canonicalW` and `row.canonicalH`. Back-compat preserved: existing 4-arg call sites see no behavior change (ratio defaults to 1.0 when `canonicalW` is omitted).

### WR-004: `tooltipId` uses `attachmentKey` for ARIA uniqueness

**Files modified:** `src/renderer/src/components/DimsBadge.tsx`
**Commit:** 0687082
**Applied fix:** Changed `tooltipId` from `` `dims-badge-tooltip-${row.attachmentName}` `` to `` `dims-badge-tooltip-${row.attachmentKey}` ``. `attachmentKey` (e.g., `default::SQUARE::SQUARE`) is unique per skin+slot+attachment combination, eliminating ARIA `id` collisions when the same attachment appears in multiple panels or multiple animation cards simultaneously.
