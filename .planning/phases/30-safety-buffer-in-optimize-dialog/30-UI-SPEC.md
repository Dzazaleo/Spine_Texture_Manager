# Phase 30 — UI Design Contract

> Visual and interaction contract for Phase 30 (Safety buffer in Optimize dialog). Generated by gsd-ui-researcher, verified by gsd-ui-checker.

> Scope is intentionally small: a single integer-percent input added to `OptimizeDialog` above the existing sharpen toggle. All design tokens, color, typography, and spacing values inherit verbatim from the project's existing Tailwind v4 token system at `src/renderer/src/index.css`. This document prescribes only what is NEW for this phase plus the locked re-use of existing tokens — no token additions, no new color, no new font sizes.

## Layout & Interaction Contract

### DOM structure (locked)

```
OptimizeDialog
├── <h2>Optimize Assets — …</h2>                            (UNCHANGED)
├── <div className="flex gap-3 mb-4">…3 summary tiles…</div> (UNCHANGED)
├── <div className="border border-border rounded-md bg-surface p-3 mb-4">  ← NEW Quality group container
│   ├── <span className="text-xs text-fg-muted mb-2 block">Quality</span>  ← NEW group header
│   ├── <label htmlFor="safety-buffer-input" className="flex items-center gap-2 mb-2 text-xs text-fg cursor-pointer">  ← NEW
│   │   ├── Safety buffer:
│   │   ├── <input id="safety-buffer-input" type="number" min={0} max={25} step={1} … title="Multiplicatively grows…">  ← NEW
│   │   └── <span className="text-fg-muted">%</span>          ← NEW
│   └── <label htmlFor="sharpen-on-export-toggle" className="flex items-center gap-2 text-xs text-fg cursor-pointer">  ← MOVED (formerly line 421-434 sibling)
│       ├── <input id="sharpen-on-export-toggle" type="checkbox" …>
│       └── Sharpen output on downscale
├── {state === 'pre-flight' && <PreFlightBody …/>}            (UNCHANGED)
├── …                                                         (UNCHANGED)
```

**Class-string discipline (Tailwind v4 Pitfall 8):** every `className` above is a single string literal. No conditional concatenation, no template strings, no dynamic class composition. The disabled-state class on the buffer input is `disabled:opacity-50 disabled:cursor-not-allowed` — appended to the literal class string verbatim, mirroring the existing sharpen toggle at line 431.

### Interaction states (locked)

| State | Trigger | Visual | Behavior |
|-------|---------|--------|----------|
| Default | dialog open in `pre-flight`, no user input yet | Input renders current `safetyBufferPercent` value (default `0`); border `border-border`; value `text-fg` | User can type / arrow / paste integer in [0, 25] |
| Focused | keyboard focus on input | Browser default `focus-visible` outline routed through `--color-accent` (orange) | Standard browser number-input behaviors active (arrow up/down step ±1, typed digits, paste) |
| Disabled | `state === 'in-progress'` | `opacity-50`, cursor `not-allowed` | All input ignored; mirrors sharpen toggle disabled treatment |
| Tooltip-visible | hover or long-focus on input | Native browser tooltip surface | `title=` attribute renders tooltip per OS/browser conventions |
| Cap-bound (silent) | `bufferCapped` flag becomes `true` for one or more rows | NO visual signal anywhere in v1.3.1 | The summary tiles (`Pages`, `Saving est.`) reflect the clamped reality; user infers cap-binding from "buffer increased but tile didn't change" |

### Reactivity contract (locked per D-08, D-11)

| Trigger | Reactive Effect |
|---------|-----------------|
| User types digit / arrow-tick / paste in input | `onChange` fires → `props.onSafetyBufferChange(clampedValue)` → AppShell `setSafetyBufferPercentLocal(value)` → OptimizeDialog re-renders → `buildExportPlan(summary, overrides, { safetyBufferPercent })` recomputes → summary tiles (`totalUsedFiles`, `toResize`, `savingsPct`) re-render with new values |
| User pastes non-numeric / decimal / out-of-range | `onChange` clamps to `[0, 25]` integer per D-04; falls back to last-valid-value if parse fails; recompute fires with the clamped value |

**Debounce:** none in v1.3.1 (D-11). Every keystroke fires a recompute.

**Validation locus (resolves CONTEXT.md "Claude's discretion #3"):** clamp lives in the `onChange` handler in `OptimizeDialog`, NOT in the AppShell setter. Rationale: keeps the input-shape contract local to the dialog (the dialog is the surface that knows about [0, 25] + integer + step 1); AppShell setter receives an already-clean value and stores it verbatim. Mirrors the existing sharpen toggle's `e.target.checked` shape: dialog produces a clean Boolean, AppShell stores it verbatim.

### Accessibility contract

| Attribute | Value | Source |
|-----------|-------|--------|
| `<input>` `id` | `safety-buffer-input` | matches existing `sharpen-on-export-toggle` id-naming convention (line 426) |
| `<label>` `htmlFor` | `safety-buffer-input` | binds label-for-input for screen readers |
| `<input>` `aria-label` | not required (label binding via `htmlFor` is sufficient) | matches existing sharpen toggle pattern |
| `<input>` `aria-describedby` | not required (tooltip via `title=` is supplementary, not required-for-comprehension) | — |
| Tab order | Quality-group safety buffer input is FIRST in the new group (above the moved sharpen toggle); both fall within the existing `useFocusTrap` cycle at `OptimizeDialog.tsx:50` | preserved by source order |
| Disabled-state announcement | native `disabled` attribute → screen readers announce as disabled | matches existing pattern |

### Behavioral edge cases (locked)

| Scenario | Behavior |
|----------|----------|
| Project has no `safetyBufferPercent` field (v1.2/v1.3-era `.stmproj`) | Input renders `0` (D-03 backward-compat) |
| User types `25` then types another `5` (would produce `255`) | `<input type="number" max={25}>` allows the typing intermediate state; `onChange` clamps to `25` on commit |
| User types `-3` | `<input type="number" min={0}>` allows the typing intermediate; `onChange` clamps to `0` on commit |
| User types `5.5` | `<input type="number" step={1}>` browser-rejects the decimal at the input level; if pasted, `onChange` floors to `5` per D-04 integer-only contract |
| User pastes `abc` | `e.target.value` parses to `NaN`; `onChange` falls back to the last-valid-value; if no prior valid value, defaults to `0` per D-04 |
| Buffer is `0` AND user toggles sharpen | Buffer multiplier path is the literal no-op per D-07; `buildExportPlan` produces byte-identical output to pre-Phase-30; no re-render of buffer-derived summary state |
| User changes buffer during `state === 'in-progress'` | Input is `disabled`; no input events fire; no recompute |
