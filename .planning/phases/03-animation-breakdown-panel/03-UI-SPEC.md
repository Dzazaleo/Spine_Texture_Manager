# Phase 3 — UI Design Contract

> Visual and interaction contract for the Animation Breakdown panel plus the
> new top-tab AppShell that hosts Phase 2's Global panel and the new panel.
> Generated by gsd-ui-researcher from CONTEXT.md (D-49..D-72) + Phase 1
> tokens (D-11..D-15) + Phase 2 table chrome (D-28..D-48) + inspected
> baseline at `src/renderer/src/panels/GlobalMaxRenderPanel.tsx`.
> Scope: `src/renderer/**` only. Per CLAUDE.md fact #5, `src/core/bones.ts`
> and `src/core/sampler.ts` extensions are pure-TS and out of UI-SPEC scope.

Phase 3 would violate D-28 and would duplicate an already-proven token set.
Registry safety gate therefore not applicable.

## Spacing Scale

Declared values (all multiples of 4 — uses Tailwind v4 default `0.25rem × n` scale):

| Token | Value | Usage in Phase 3 |
|-------|-------|-------|
| 0.5 (2px)  | 2px  | Sort-arrow pixel nudge only (inherited from Phase 2) |
| 1 (4px)    | 4px  | `gap-1` between tab buttons, `px-0.5` on `<mark>` highlight |
| 2 (8px)    | 8px  | `py-2` vertical row padding (inherited from Phase 2 D-47) |
| 3 (12px)   | 12px | `px-3` horizontal cell padding (inherited from Phase 2 D-47); `gap-3` between card list items |
| 4 (16px)   | 16px | `mb-4` between panel header and card list; `gap-4` in panel header |
| 6 (24px)   | 24px | `px-6` AppShell header horizontal padding |
| 8 (32px)   | 32px | `p-8` panel body padding (inherited from Phase 2); `py-8` empty-state row |

**Row height:** `py-2 px-3` → ~32px per row, mirroring Phase 2's table (D-47).
**Card spacing:** `gap-3` (12px) between cards in the card list — tighter than
section-level spacing to communicate "these are siblings in a scrollable list,"
but enough breathing room that headers are visually distinct when multiple are
expanded.

**Permitted 2px exceptions (closed set — grep-verifiable acceptance):**
- Chip padding `py-0.5` (2px) on the Source Animation chip (inherited from Phase 2 D-44).
- Chip padding `py-0.5` (2px) on the (disabled) Override button — reuses the same Phase 2 chip shape.
- Chip padding `py-0.5` (2px) on the filename chip hoisted into AppShell header (inherited from Phase 2 D-44 chip class verbatim).
- Match-highlight `<mark>` padding `px-0.5` (2px horizontal) (inherited from Phase 2 D-40).
- Sort-arrow pixel nudge (inherited from Phase 2).

**NO other 2px use allowed in Phase 3.** Any new `py-0.5` / `px-0.5` / `p-0.5`
beyond the five uses listed above is a contract violation and must be rewritten
to the 4/8/12/16/24/32 scale.

## Typography

Three sizes, exactly two weights. All monospace inside the panel body (Phase 2
D-47 carries forward). System-sans retained only for AppShell chrome headings.

| Role | Size | Weight | Line Height | Tailwind class | Usage |
|------|------|--------|-------------|----------------|-------|
| Panel heading | 18px | 600 (semibold) | 1.4 | `text-lg font-semibold` | "Animation Breakdown" panel header (one per panel body) |
| Card header | 14px | 600 (semibold) | 1.5 | `text-sm font-semibold font-mono` | Collapsed/expanded card title row (`▸ {name} — N unique assets…`) |
| Table header | 12px | 600 (semibold) | 1.5 | `text-xs font-semibold font-mono` | Column labels inside an expanded card (inherited from Phase 2 D-47) |
| Row cell | 14px | 400 (regular) | 1.5 | `text-sm font-mono` | Per-row cells (inherited from Phase 2 D-47) |
| Bone Path cell | 12px | 400 (regular) | 1.5 | `text-xs font-mono text-fg-muted` | De-emphasized structural info (CONTEXT D-67) |
| Tab button (active) | 14px | 600 (semibold) | 1.5 | `text-sm font-semibold` | Active tab label (`Global` or `Animation Breakdown`) |
| Tab button (inactive) | 14px | 400 (regular) | 1.5 | `text-sm font-normal` | Inactive tab label |
| Empty state row | 14px | 400 (regular) | 1.5 | `text-sm font-mono text-fg-muted text-center` | "No assets referenced" / "No attachments match…" (CONTEXT D-62, inherits Phase 2 D-41) |

**Weights:** exactly two — regular (400) + semibold (600). No medium (500)
anywhere in Phase 3. Active/inactive tab contrast is carried by **color**
(`text-accent` vs `text-fg-muted`) plus the **2px `bg-accent` underline
indicator** on the active tab — both high-signal channels that dominate the
weight delta anyway. Dropping medium matches the rest of the chrome convention
(card headers + table headers all use 600; everything else 400).

**Line heights:** 1.5 everywhere (Tailwind default); 1.4 on the one `text-lg`
heading. No `text-2xl`/`text-xl`/`text-4xl` allowed in Phase 3.

**Forbidden classes (grep-verifiable negative acceptance):** `font-medium`,
`font-weight: 500`, any custom CSS `font-weight` rule between 400 and 600.

## Interaction Contracts

Prescriptive resolutions for every CONTEXT "Claude's Discretion" item. Each
decision is grep-verifiable via the cited class signature or exact constant.

### AppShell tab strip visual style → **underline indicator**

| Element | Specification |
|---------|---------------|
| Tab strip container | `<nav role="tablist" className="flex gap-1 items-center">` |
| Tab button (base, shared) | `<button role="tab" type="button" aria-selected={isActive} className="relative px-4 py-2 text-sm font-sans transition-colors focus:outline-none focus-visible:outline-2 focus-visible:outline-accent">` — note: NO weight class on the base; weight is composed per-state below. |
| Tab label weight + color (inactive) | `font-normal text-fg-muted hover:text-fg` (weight 400) |
| Tab label weight + color (active) | `font-semibold text-accent` (weight 600) — Phase 1 D-14 accent usage list item: the active tab label is a _reserved_ element for accent color. |
| Active underline indicator | An absolutely-positioned `<span aria-hidden className="absolute left-0 right-0 -bottom-px h-[2px] bg-accent" />` inside the active tab button only. The 2px height sits on the AppShell header's bottom `border-border` (1px), visually "cutting through" the border to anchor the tab to the content below. Rationale: matches dev-tool panels (Chrome DevTools, VSCode); pill-style active states read as clickable chip-rows rather than sticky navigation; orange under a low-saturation warm-stone chrome gives unambiguous "this is where you are" signal without over-rendering. |

**Active/inactive contrast budget:** three orthogonal channels carry the
active-state signal — **weight** (600 vs 400) + **color** (`text-accent` vs
`text-fg-muted`) + **underline** (2px `bg-accent` span vs nothing). Removing
the medium-weight intermediate step keeps the contract at exactly two weights
project-wide without weakening the signal, because the remaining two channels
(color + underline) already give an unambiguous state distinction.

**Rejected:** pill-style background fills like `bg-accent/10 rounded-md` —
reads as a hover state on other accent-tinted surfaces elsewhere in the app
(selected rows, match highlights). Underline is unambiguous chrome.

### Flash-highlight on jump-target card → **900ms ring-2 ring-accent, linear fade-out**

| Element | Specification |
|---------|---------------|
| Trigger | AppShell sets `focusAnimationName`; AnimationBreakdownPanel `useEffect` on `[focusAnimationName]` adds the target `cardId` to `userExpanded`, scrolls the card into view, and sets a local `isFlashing === cardId` state. |
| Flash applied via | `ring-2 ring-accent ring-offset-2 ring-offset-surface` on the card outer `<section>` wrapper **only when `isFlashing === cardId`**; class toggled via `clsx` conditional. |
| Duration | **900ms** (`setTimeout(() => { setIsFlashing(null); onFocusConsumed(); }, 900)`). Rationale: 800ms is the floor perceptual-wise for "something happened, look here"; 1200ms starts to feel slow after rapid successive jumps. 900ms threads the needle. |
| Easing | No transition on ring appear (instant); ring disappears instantly when the 900ms timeout clears. No pulse / no fade. Rationale: a pulse reads as an error toast; a fade looks unfinished. Instant on/off is decisive. |
| Scroll behavior | `element.scrollIntoView({ behavior: 'smooth', block: 'start' })` — smooth scroll confirms navigation; `block: 'start'` anchors the flashing card header at the top of the viewport. |
| Cleanup | `onFocusConsumed()` fires at the end of the 900ms window; AppShell clears its `focusAnimationName` so manual nav state stays clean (CONTEXT D-66). |

**Rejected:** `bg-accent/20` pulse — conflicts with match-highlight `<mark>`
background (same token, viewer conflates the two signals). `ring-2 ring-accent`
is visually distinct from row-level `bg-accent/5` selection and from the
match-highlight.

### Collapsible card ARIA pattern → **hand-rolled `<button aria-expanded>`**

| Element | Specification |
|---------|---------------|
| Card wrapper | `<section aria-labelledby={headerId} className="border border-border rounded-md bg-panel overflow-hidden">` — `overflow-hidden` so the expand/collapse does not leak the table's bottom border outside the card rounded radius. |
| Card header | `<button id={headerId} type="button" aria-expanded={isExpanded} aria-controls={bodyId} onClick={onToggle} className="w-full flex items-center gap-2 px-4 py-3 text-left text-sm font-semibold font-mono text-fg hover:bg-accent/5 focus:outline-none focus-visible:outline-2 focus-visible:outline-accent">` |
| Caret glyph | `▸` (U+25B8) when collapsed / `▾` (U+25BE) when expanded — rendered as the first `<span>` inside the button, `text-fg-muted` (caret is structural, not semantic). |
| Card body | `<div id={bodyId} role="region" className="border-t border-border">{expanded && <Table … />}</div>` — the body is only rendered when `isExpanded === true` (no `display: none` / CSS-hidden) to keep the DOM cheap per CONTEXT D-63. |
| Keyboard activation | Native `<button>` — Space + Enter both toggle. No custom keydown handler needed. |

**Default expand state (grep-verifiable initial literal, CONTEXT D-63 + D-64):**
AnimationBreakdownPanel MUST initialize `userExpanded` with the exact literal
`new Set(['setup-pose'])` — Setup Pose card expanded by default, all animation
cards collapsed. Executor grep target: `new Set(['setup-pose'])` (single-quoted)
or `new Set(["setup-pose"])` (double-quoted) — either quoting style is
acceptable; the literal string token `setup-pose` and the `new Set(` construct
are the required signatures.

**Rejected:** native `<details>/<summary>` — gives free keyboard support but
sacrifices tight CSS control over the header row and forces a browser-specific
default marker that fights the design. The hand-rolled pattern is ~30 extra
lines and matches Phase 2's hand-rolled discipline.

### Bone Path truncation → **hand-rolled mid-ellipsis**

| Element | Specification |
|---------|---------------|
| Full path format | `root → CTRL → CHAIN_2 → CHAIN_3 → … → CHAIN_8 → {slotName} → {attachmentName}` |
| Cell wrapper | `<td title={fullBonePath} className="py-2 px-3 font-mono text-xs text-fg-muted max-w-[320px]">` — fixed-max width caps the column; `max-w-[320px]` is the column budget. |
| Truncation routine | Hand-rolled function `truncateMidEllipsis(path: string[], maxChars: number): string` — if `path.join(' → ').length <= maxChars`, return joined; else keep `path[0]` and the last 2 tokens (`path.slice(-2)`), insert `…` in the middle, re-join. One function, pure, testable. Tests: `tests/renderer/bone-path.spec.ts` (add under existing vitest config). |
| `maxChars` threshold | **48** — rationale: `max-w-[320px]` at 12px JetBrains Mono renders ~45–48 characters reliably; 48 is the conservative ceiling that never overflows even with capital letters. Executor may tune during human-verify if real rigs overflow. |
| Hover reveal | Native browser tooltip via `title={fullBonePath}` — consistent with Phase 2 D-44 chip styling (no custom tooltip library introduced). |
| Empty Bone Path fallback | If `path.length === 0` (should never happen — root always exists), render the empty string. Do not render a placeholder glyph. |

**Rejected:** CSS-only `text-overflow: ellipsis` (end-ellipsis) — loses the
attachment-name leaf, which is the most actionable token in the path. The
user needs to see "→ …→ CHAIN_8 → CIRCLE" not "root → CTRL → CHAIN_2 → CHAIN…".

### Source Animation chip → jump-button upgrade (D-72 resolution)

Locked class string for the upgraded cell in `GlobalMaxRenderPanel.tsx`:

```tsx
<button
  type="button"
  onClick={() => onJumpToAnimation(row.sourceLabel)}
  aria-label={`Jump to ${row.sourceLabel} in Animation Breakdown`}
  className="inline-block border border-border rounded-md px-2 py-0.5 text-xs font-mono text-fg cursor-pointer hover:bg-accent/10 focus:outline-none focus-visible:outline-2 focus-visible:outline-accent"

</button>
```

- Shape (border / rounded / padding / font) is identical to Phase 2 D-44's
  `<span>` chip — pure markup swap, no layout shift.
- `hover:bg-accent/10` is a **new** background-tint level (Phase 3 introduces
  it for this single element; codifies "this chip is now clickable"). It sits
  between `bg-accent/5` (row hover / selection) and `bg-accent/20` (match
  highlight) so the three accent-alpha levels communicate increasing intensity:
  row-ambient → interactive-hover → active-match.
- The click fires `onJumpToAnimation(row.sourceLabel)`; for Setup Pose rows
  the callback receives the literal `'Setup Pose (Default)'` which the target
  panel interprets as the `setup-pose` cardId (CONTEXT D-72).
- Non-interactive fallback: if the `onJumpToAnimation` prop is undefined
  (component rendered outside AppShell), render a non-button `<span>` with the
  original Phase 2 chip class — preserves decoupling.
