<objective>
Add the per-project `loaderMode: 'auto' | 'atlas-less'` field to the `.stmproj` v1 schema with full validate/migrate/materialize/serialize plumbing, plus thread it through `SamplerWorkerData` and the two `loadSkeleton` invocation sites in `src/main/project-io.ts`. The Phase 20 documentation pre-massage pattern is the direct template (RESEARCH.md §Pitfall 6).

Purpose: Closes the persistence half of D-08. Without this plan, the user can flip the toggle in the renderer (Plan 08) but the choice doesn't survive Save → Reopen. The sampler-worker threading is critical because the worker re-loads the skeleton in-process — without `loaderMode` in `SamplerWorkerData`, the worker would re-canonicalize a project the user explicitly marked atlas-less.

The recovery path (`handleProjectReloadWithSkeleton`, project-io.ts:623-720) DOES NOT have access to a materialized project at the branch where loadSkeleton is called — it only sees the primitive args object posted by the renderer. To preserve D-08 through the recovery flow, this plan widens the recovery args interface to accept `loaderMode?: 'auto' | 'atlas-less'` from the renderer (which holds the materialized loaderMode in its useState slot per Plan 08), and threads it into the recovery loadSkeleton + runSamplerInWorker calls. The renderer-side wiring of this field is Plan 08's responsibility (the renderer simply adds `loaderMode` to the recovery dispatch payload alongside the existing `mergedOverrides`/`samplingHz`/etc. fields).
